## 1.引言

在实际工作中，我们经常需要将 SQL 文件或数据文件导入到 MySQL 数据库中，用于数据恢复、迁移或初始化数据库。

本文介绍四种常用的数据导入方法：使用 `mysql` 命令导入 SQL 文件、使用 `source` 命令在 MySQL 命令行中导入、使用 `LOAD DATA` 语句导入文本文件，以及使用 `mysqlimport` 工具导入数据。通过实际示例，你将学会如何选择合适的导入方式，并掌握各种导入场景的操作方法。

## 2.使用 mysql 命令导入

### 2.1.什么是 mysql 命令导入？

`mysql` 命令是 MySQL 提供的命令行客户端工具，可以直接将 SQL 文件导入到数据库中。这是最常用的导入方式，适合导入包含完整 SQL 语句的文件。

**基本语法**：


```bash
mysql -u username -p database_name < file.sql
```

**参数说明**：

- `-u username`：MySQL 用户名
- `-p`：提示输入密码
- `database_name`：目标数据库名称
- `< file.sql`：要导入的 SQL 文件路径

### 2.2.基本导入示例

假设我们有一个名为 `shop_backup.sql` 的备份文件，需要导入到 `shop_db` 数据库中：


```bash
# 使用 root 用户导入数据库备份文件
mysql -u root -p shop_db < shop_backup.sql
```

**执行过程**：

1. 系统会提示输入 MySQL 用户密码
2. 输入密码后按回车键
3. MySQL 会读取 SQL 文件并执行其中的所有语句
4. 导入完成后返回命令提示符

### 2.3.指定主机和端口

如果 MySQL 服务器不在本地，或者使用了非默认端口，需要指定主机地址和端口号：


```bash
# 连接到远程 MySQL 服务器并导入数据
mysql -u admin -p -h 192.168.1.100 -P 3306 shop_db < shop_backup.sql
```

**参数说明**：

- `-h 192.168.1.100`：指定 MySQL 服务器的主机地址
- `-P 3306`：指定 MySQL 服务器的端口号（注意是大写的 P）

### 2.4.导入注意事项

使用 `mysql` 命令导入时需要注意以下几点：

1. **数据库必须存在**：在导入之前，确保目标数据库已经创建。如果数据库不存在，需要先创建：


   ```sql
   CREATE DATABASE shop_db;
   ```

2. **文件路径**：确保 SQL 文件的路径正确，可以使用绝对路径或相对路径

3. **文件编码**：确保 SQL 文件的字符编码正确，建议使用 UTF-8 编码

4. **权限要求**：执行导入的用户需要有相应的数据库操作权限

## 3.使用 source 命令导入

### 3.1.什么是 source 命令？

`source` 命令是 MySQL 命令行客户端内置的命令，可以在 MySQL 命令行环境中直接执行 SQL 文件。这种方式不需要退出 MySQL 命令行，使用起来更加方便。

**基本语法**：


```sql
source /path/to/file.sql;
```

### 3.2.基本导入步骤

使用 `source` 命令导入数据需要先登录到 MySQL 命令行：

**步骤一：登录 MySQL**


```bash
# 使用 root 用户登录 MySQL
mysql -u root -p
```

**步骤二：创建数据库（如果不存在）**


```sql
# 创建名为 user_db 的数据库
CREATE DATABASE user_db;
```

**步骤三：选择数据库**


```sql
# 切换到 user_db 数据库
USE user_db;
```

**步骤四：设置字符编码**


```sql
# 设置客户端字符编码为 UTF-8
SET NAMES utf8mb4;
```

**步骤五：导入 SQL 文件**


```sql
# 导入备份文件（使用绝对路径）
source /Users/data/user_backup.sql;
```

### 3.3.使用相对路径导入

如果 SQL 文件在当前工作目录中，也可以使用相对路径：


```sql
# 导入当前目录下的 SQL 文件
source ./user_backup.sql;
```

### 3.4.source 命令的优势

使用 `source` 命令的好处：

- 不需要退出 MySQL 命令行
- 可以在导入前后执行其他 SQL 命令
- 适合需要分步骤操作的场景
- 可以直接看到导入过程中的错误信息

## 4.使用 LOAD DATA 导入数据

### 4.1.什么是 LOAD DATA？

`LOAD DATA` 是 MySQL 提供的 SQL 语句，用于将文本文件中的数据快速导入到数据表中。这种方式适合导入 CSV、TXT 等格式的数据文件，导入速度通常比逐条 INSERT 语句快很多。

**基本语法**：


```sql
LOAD DATA [LOCAL] INFILE 'file_path' 
INTO TABLE table_name;
```

### 4.2.基本导入示例

假设我们有一个 `product_data.txt` 文件，需要导入到 `product_table` 表中：


```sql
# 从本地文件导入数据到 product_table 表
LOAD DATA LOCAL INFILE 'product_data.txt' INTO TABLE product_table;
```

**参数说明**：

- `LOCAL`：指定从客户端主机读取文件（不加 LOCAL 则从服务器读取）
- `'product_data.txt'`：数据文件路径
- `INTO TABLE product_table`：目标表名

### 4.3.指定字段分隔符

默认情况下，`LOAD DATA` 使用制表符（Tab）作为字段分隔符。如果数据文件使用其他分隔符，需要指定：


```sql
# 使用逗号作为字段分隔符，Windows 换行符作为行分隔符
LOAD DATA LOCAL INFILE 'product_data.txt' 
INTO TABLE product_table
FIELDS TERMINATED BY ','
LINES TERMINATED BY '\r\n';
```

**参数说明**：

- `FIELDS TERMINATED BY ','`：字段之间使用逗号分隔
- `LINES TERMINATED BY '\r\n'`：行之间使用 Windows 换行符分隔

### 4.4.指定字段包围符

如果数据文件中的字段被引号包围，需要指定包围符：


```sql
# 字段用双引号包围，使用逗号分隔
LOAD DATA LOCAL INFILE 'customer_data.csv' 
INTO TABLE customer_table
FIELDS TERMINATED BY ',' 
ENCLOSED BY '"'
LINES TERMINATED BY '\n';
```

### 4.5.指定列顺序

如果数据文件中的列顺序与表中的列顺序不一致，需要明确指定列的对应关系：


```sql
# 数据文件列顺序为：name, price, id
# 表中列顺序为：id, name, price
# 需要指定列的对应关系
LOAD DATA LOCAL INFILE 'order_data.txt' 
INTO TABLE order_table (name, price, id);
```

### 4.6.跳过表头行

如果数据文件的第一行是表头（列名），需要跳过：


```sql
# 跳过第一行（表头）
LOAD DATA LOCAL INFILE 'sales_data.csv' 
INTO TABLE sales_table
FIELDS TERMINATED BY ','
LINES TERMINATED BY '\n'
IGNORE 1 LINES;
```

### 4.7.注意事项

使用 `LOAD DATA` 时需要注意：

- 如果使用 `LOCAL`，文件必须在客户端机器上
- 如果不使用 `LOCAL`，文件必须在 MySQL 服务器上，且需要 FILE 权限
- 确保数据文件的格式与表结构匹配
- 导入前建议先备份表数据

## 5.使用 mysqlimport 导入数据

### 5.1.什么是 mysqlimport？

`mysqlimport` 是 MySQL 提供的命令行工具，它实际上是 `LOAD DATA INFILE` 语句的命令行接口。使用 `mysqlimport` 可以更方便地从命令行导入数据文件。

**基本语法**：


```bash
mysqlimport [选项] database_name file_name
```

### 5.2.基本导入示例

假设我们有一个 `order_data.txt` 文件，需要导入到 `shop_db` 数据库的 `order_table` 表中：


```bash
# 从本地文件导入数据到 order_table 表
mysqlimport -u root -p --local shop_db order_data.txt
```

**参数说明**：

- `-u root`：MySQL 用户名
- `-p`：提示输入密码
- `--local`：从客户端主机读取文件
- `shop_db`：数据库名称
- `order_data.txt`：数据文件名（注意：文件名必须与表名相同）

**重要提示**：`mysqlimport` 要求文件名必须与目标表名相同。例如，要导入到 `order_table` 表，文件名必须是 `order_table.txt`。

### 5.3.指定字段分隔符

使用 `--fields-terminated-by` 选项指定字段分隔符：


```bash
# 使用冒号作为字段分隔符
mysqlimport -u root -p --local \
  --fields-terminated-by=':' \
  shop_db order_table.txt
```

### 5.4.指定行分隔符

使用 `--lines-terminated-by` 选项指定行分隔符：


```bash
# 指定字段和行分隔符
mysqlimport -u root -p --local \
  --fields-terminated-by=',' \
  --lines-terminated-by='\r\n' \
  shop_db order_table.txt
```

### 5.5.指定列顺序

如果数据文件的列顺序与表结构不同，使用 `--columns` 选项指定：


```bash
# 指定列的导入顺序
mysqlimport -u root -p --local \
  --columns=name,price,id \
  shop_db order_table.txt
```

### 5.6.常用选项说明

`mysqlimport` 提供了许多有用的选项：

| 选项                          | 功能说明                       |
| ----------------------------- | ------------------------------ |
| `-d` 或 `--delete`            | 导入前删除表中的所有数据       |
| `-f` 或 `--force`             | 遇到错误时继续执行             |
| `-i` 或 `--ignore`            | 忽略重复键错误，跳过冲突的记录 |
| `-r` 或 `--replace`           | 遇到重复键时替换现有记录       |
| `-l` 或 `--lock-tables`       | 导入前锁定表                   |
| `--fields-enclosed-by=char`   | 指定字段包围符                 |
| `--fields-terminated-by=char` | 指定字段分隔符                 |
| `--lines-terminated-by=str`   | 指定行分隔符                   |
| `-v`                          | 显示详细执行信息               |
| `-p`                          | 提示输入密码                   |

### 5.7.实际应用示例

**示例一：导入前清空表**


```bash
# 导入前删除表中所有数据，然后导入新数据
mysqlimport -u root -p --local --delete shop_db product_table.txt
```

**示例二：忽略重复记录**


```bash
# 如果遇到重复的主键，跳过该记录继续导入
mysqlimport -u root -p --local --ignore shop_db product_table.txt
```

**示例三：替换重复记录**


```bash
# 如果遇到重复的主键，用新数据替换旧数据
mysqlimport -u root -p --local --replace shop_db product_table.txt
```

## 6.导入方法对比

### 6.1.适用场景

**mysql 命令**：

- 适合导入完整的 SQL 备份文件
- 包含 CREATE TABLE、INSERT 等完整 SQL 语句
- 用于数据库恢复和迁移

**source 命令**：

- 适合在 MySQL 命令行环境中操作
- 需要分步骤执行时使用
- 方便查看导入过程中的错误

**LOAD DATA**：

- 适合导入纯数据文件（CSV、TXT）
- 导入速度快，适合大量数据
- 需要数据文件格式规范

**mysqlimport**：

- 适合从命令行快速导入数据文件
- 文件名必须与表名相同
- 提供了丰富的选项控制导入行为

### 6.2.性能对比

- **LOAD DATA** 和 **mysqlimport**：导入速度最快，适合大量数据
- **mysql 命令**：速度中等，需要解析 SQL 语句
- **source 命令**：速度与 mysql 命令相同，但更灵活

## 7.实践建议

### 7.1.导入前准备

1. **备份现有数据**：导入前先备份目标数据库或表
2. **检查文件格式**：确保数据文件格式正确，编码为 UTF-8
3. **验证表结构**：确保目标表已创建，且结构匹配
4. **检查磁盘空间**：确保有足够的磁盘空间

### 7.2.选择合适的导入方法

- **SQL 备份文件**：使用 `mysql` 命令或 `source` 命令
- **CSV/TXT 数据文件**：使用 `LOAD DATA` 或 `mysqlimport`
- **需要交互操作**：使用 `source` 命令
- **批量自动化**：使用 `mysql` 命令或 `mysqlimport`

### 7.3.错误处理

- 导入过程中如果出现错误，检查错误信息
- 确保数据文件格式与表结构匹配
- 检查字符编码是否正确
- 验证用户权限是否足够

### 7.4.大文件导入

对于大型数据文件：

- 使用 `LOAD DATA` 或 `mysqlimport` 提高速度
- 考虑分批导入
- 导入时关闭不必要的索引，导入后重建
- 监控导入进度和系统资源

## 8.总结

本文介绍了 MySQL 数据导入的四种主要方法：

1. **mysql 命令**：适合导入完整的 SQL 备份文件
2. **source 命令**：在 MySQL 命令行中直接执行 SQL 文件
3. **LOAD DATA**：快速导入文本格式的数据文件
4. **mysqlimport**：命令行工具，提供丰富的导入选项

**关键要点**：

- 根据数据文件类型选择合适的导入方法
- 导入前做好数据备份
- 确保文件格式和表结构匹配
- 注意字符编码和权限设置

掌握这些导入方法后，你可以灵活地进行数据恢复、迁移和初始化操作，为数据库管理提供有力支持。
