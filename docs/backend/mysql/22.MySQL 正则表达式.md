## 1.引言

在数据库查询中，我们经常需要查找符合特定模式的字符串，而不仅仅是简单的模糊匹配。

**实际场景举例**：

- 查找所有符合邮箱格式的记录（如 `user@example.com`、`admin@company.net`）
- 查找所有以数字开头的产品编号（如 `2024PROD001`、`123ABC`）
- 查找所有包含特定格式的订单号（如 `ORD-2024-001`、`ORD-2024-999`）

虽然 LIKE 子句可以进行简单的模糊匹配，但对于更复杂的模式匹配需求，我们需要使用更强大的工具——正则表达式。

**正则表达式的作用**：
正则表达式是一种强大的模式匹配工具，它允许我们使用特殊的字符和语法来描述字符串的模式。MySQL 提供了 `REGEXP` 和 `RLIKE` 操作符来支持正则表达式匹配，这使得我们可以进行更精确、更灵活的字符串查询。

本文将从正则表达式的基础概念开始，逐步介绍如何在 MySQL 中使用正则表达式进行模式匹配，包括各种正则模式的含义、REGEXP 和 RLIKE 操作符的使用方法，以及实际应用示例。

## 2.正则表达式基础概念

### 2.1.什么是正则表达式？

正则表达式（Regular Expression，简称 regex）是一种用于描述字符串模式的特殊语法。它使用一系列特殊字符和普通字符来定义一个搜索模式，可以用来检查字符串是否符合某种格式、查找符合模式的字符串，或者替换字符串中的特定部分。

**类比理解**：

- 普通查询（使用 `=`）就像精确查找，必须完全匹配
- LIKE 查询（使用 `%` 和 `_`）就像简单的模糊搜索，可以匹配部分内容
- 正则表达式就像高级搜索，可以定义复杂的匹配规则，比如"以字母开头，后跟3个数字，再跟一个连字符"

### 2.2.MySQL 中的正则表达式

MySQL 支持正则表达式匹配，主要通过两个操作符实现：

- **`REGEXP`**：用于正则表达式匹配
- **`RLIKE`**：与 `REGEXP` 功能完全相同，可以互换使用

这两个操作符的作用完全一致，选择使用哪一个主要取决于个人习惯或团队规范。

### 2.3.正则表达式与 LIKE 的区别

虽然 LIKE 和正则表达式都可以进行模式匹配，但它们的能力和适用场景有所不同：

| 特性     | LIKE                              | REGEXP/RLIKE             |
| -------- | --------------------------------- | ------------------------ |
| 通配符   | `%`（任意字符）和 `_`（单个字符） | 丰富的正则语法           |
| 匹配能力 | 简单模式匹配                      | 复杂模式匹配             |
| 性能     | 在某些情况下可以使用索引          | 通常无法使用索引         |
| 适用场景 | 简单的模糊搜索                    | 复杂的格式验证和模式匹配 |

**示例对比**：

- 使用 LIKE 查找以 "A" 开头的产品：`WHERE product_code LIKE 'A%'`
- 使用 REGEXP 查找以字母开头、后跟数字的产品：`WHERE product_code REGEXP '^[A-Za-z][0-9]'`

## 3.正则表达式基本语法

### 3.1.基本匹配字符

在正则表达式中，大部分普通字符（字母、数字）会直接匹配它们自己。

**示例**：

- 模式 `'abc'` 会匹配包含 "abc" 的字符串，如 "abc"、"xabc"、"abc123"
- 模式 `'123'` 会匹配包含 "123" 的字符串，如 "123"、"abc123"、"123xyz"

### 3.2.特殊字符和元字符

正则表达式的强大之处在于它支持各种特殊字符（元字符），这些字符具有特殊含义：

| 元字符 | 名称       | 功能说明                             |
| ------ | ---------- | ------------------------------------ |
| `^`    | 脱字符     | 匹配字符串的开始位置                 |
| `$`    | 美元符     | 匹配字符串的结束位置                 |
| `.`    | 点号       | 匹配除换行符外的任意单个字符         |
| `*`    | 星号       | 匹配前面的元素零次或多次             |
| `+`    | 加号       | 匹配前面的元素一次或多次             |
| `?`    | 问号       | 匹配前面的元素零次或一次             |
| `|`    | 竖线       | 表示"或"，匹配多个模式之一           |
| `[]`   | 方括号     | 字符集合，匹配方括号内的任意一个字符 |
| `[^]`  | 否定字符集 | 匹配不在方括号内的任意字符           |
| `{}`   | 花括号     | 指定匹配次数                         |
| `()`   | 圆括号     | 分组，将多个元素组合在一起           |

## 4.常用正则表达式模式

### 4.1.位置匹配

#### 4.1.1.匹配字符串开始（^）

`^` 符号用于匹配字符串的开始位置。

**示例**：


```sql
SELECT * FROM customers WHERE customer_name REGEXP '^张';
```

这个查询会匹配所有以 "张" 开头的客户姓名，如 "张三"、"张伟"、"张明"。

**注意**：`^` 必须放在模式的最前面才表示字符串开始。如果放在其他位置，它就是一个普通字符。

#### 4.1.2.匹配字符串结束（$）

`$` 符号用于匹配字符串的结束位置。

**示例**：


```sql
SELECT * FROM products WHERE product_code REGEXP '001$';
```

这个查询会匹配所有以 "001" 结尾的产品编号，如 "PROD001"、"ITEM001"、"2024001"。

**组合使用**：


```sql
SELECT * FROM products WHERE product_code REGEXP '^PROD.*001$';
```

这个查询匹配以 "PROD" 开头、以 "001" 结尾的产品编号，中间可以有任意字符。

### 4.2.字符匹配

#### 4.2.1.匹配任意字符（.）

`.` 符号可以匹配除换行符（`\n`）外的任意单个字符。

**示例**：


```sql
SELECT * FROM users WHERE username REGEXP '^a.b$';
```

这个查询会匹配：

- `a1b`（中间是数字）
- `axb`（中间是字母）
- `a_b`（中间是下划线）
- `a b`（中间是空格）

但不会匹配：

- `ab`（中间没有字符）
- `a12b`（中间有两个字符）

#### 4.2.2.字符集合（[]）

方括号 `[]` 用于定义一个字符集合，匹配方括号内的任意一个字符。

**示例**：


```sql
SELECT * FROM products WHERE product_code REGEXP '^[ABC]';
```

这个查询会匹配所有以 "A"、"B" 或 "C" 开头的产品编号。

**字符范围**：
可以使用连字符 `-` 表示字符范围：

- `[0-9]`：匹配任意数字（0 到 9）
- `[a-z]`：匹配任意小写字母（a 到 z）
- `[A-Z]`：匹配任意大写字母（A 到 Z）
- `[A-Za-z]`：匹配任意字母（大小写均可）

**示例**：


```sql
SELECT * FROM orders WHERE order_number REGEXP '^[A-Z][0-9]';
```

这个查询匹配以大写字母开头、后跟一个数字的订单号，如 "A1"、"B5"、"Z9"。

#### 4.2.3.否定字符集（[^]）

在方括号内，`^` 放在开头表示"否定"，匹配不在方括号内的任意字符。

**示例**：


```sql
SELECT * FROM products WHERE product_code REGEXP '^[^0-9]';
```

这个查询会匹配所有不以数字开头的产品编号，即第一个字符必须是字母或符号。

### 4.3.数量限定符

#### 4.3.1.星号（*）

`*` 表示匹配前面的元素零次或多次。

**示例**：


```sql
SELECT * FROM products WHERE description REGEXP '优惠*';
```

这个查询会匹配：

- `优`（"惠"出现0次）
- `优惠`（"惠"出现1次）
- `优惠惠`（"惠"出现2次）
- `优惠惠惠`（"惠"出现3次）
- `优惠惠惠惠`（"惠"出现多次）

**注意**：`*` 匹配的是它前面的**单个元素**，所以 `'优惠*'` 表示 "优" 后面跟零个或多个 "惠"。如果要匹配整个词 "优惠" 的重复，需要使用分组：`'(优惠)*'`。

#### 4.3.2.加号（+）

`+` 表示匹配前面的元素一次或多次（至少一次）。

**示例**：


```sql
SELECT * FROM orders WHERE order_id REGEXP '^ORD-[0-9]+$';
```

这个查询会匹配：

- `ORD-123`（包含一个或多个数字，这里是3个数字）
- `ORD-12345`（包含一个或多个数字，这里是5个数字）
- `ORD-1`（包含一个或多个数字，这里是1个数字）

但不会匹配：

- `ORD-`（没有数字）
- `ORD-ABC`（不是数字）

#### 4.3.3.问号（?）

`?` 表示匹配前面的元素零次或一次（可选）。

**示例**：


```sql
SELECT * FROM products WHERE product_code REGEXP '^PROD-?[0-9]+$';
```

这个查询会匹配：

- `PROD-123`（有连字符）
- `PROD123`（没有连字符）

#### 4.3.4.花括号（）

花括号用于指定匹配次数：

- `{n}`：恰好匹配 n 次
- `{n,}`：至少匹配 n 次
- `{n,m}`：匹配 n 到 m 次

**示例**：


```sql
-- 匹配恰好 3 个数字
SELECT * FROM products WHERE product_code REGEXP '^[A-Z]{3}[0-9]{3}$';

-- 匹配至少 2 个数字
SELECT * FROM products WHERE product_code REGEXP '^[A-Z]+[0-9]{2,}$';

-- 匹配 2 到 4 个数字
SELECT * FROM products WHERE product_code REGEXP '^[A-Z]+[0-9]{2,4}$';
```

### 4.4.逻辑操作符

#### 4.4.1.或操作符（|）

竖线 `|` 表示"或"，匹配多个模式之一。

**示例**：


```sql
SELECT * FROM employees WHERE department REGEXP '技术|销售|人事';
```

这个查询会匹配部门为 "技术"、"销售" 或 "人事" 的员工。

**使用分组**：


```sql
SELECT * FROM products WHERE product_name REGEXP '^(手机|电脑|平板)';
```

这个查询匹配以 "手机"、"电脑" 或 "平板" 开头的产品名称。

### 4.5.预定义字符类

MySQL 的正则表达式支持一些预定义的字符类：

| 字符类      | 说明       | 等价写法        |
| ----------- | ---------- | --------------- |
| `[:alnum:]` | 字母和数字 | `[A-Za-z0-9]`   |
| `[:alpha:]` | 字母       | `[A-Za-z]`      |
| `[:digit:]` | 数字       | `[0-9]`         |
| `[:lower:]` | 小写字母   | `[a-z]`         |
| `[:upper:]` | 大写字母   | `[A-Z]`         |
| `[:space:]` | 空白字符   | `[ \t\n\r\f\v]` |

**示例**：


```sql
SELECT * FROM users WHERE username REGEXP '^[[:alpha:]][[:alnum:]]*$';
```

这个查询匹配以字母开头、后跟字母或数字的用户名。

## 5.REGEXP 操作符的使用

### 5.1.基本语法

REGEXP 操作符用于在 WHERE 子句中进行正则表达式匹配，基本语法如下：


```sql
SELECT column1, column2, ...
FROM table_name
WHERE column_name REGEXP 'pattern';
```

**参数说明**：

- `column1, column2, ...`：要查询的列名，使用 `*` 表示选择所有列
- `table_name`：要查询的数据表名称
- `column_name`：要进行正则表达式匹配的列名
- `'pattern'`：正则表达式模式

### 5.2.基本匹配示例

#### 5.2.1.查找以特定字符串开头的记录

查找客户表中所有姓名以 "王" 开头的客户：


```sql
SELECT * FROM customers WHERE customer_name REGEXP '^王';
```

#### 5.2.2.查找以特定字符串结尾的记录

查找产品表中所有编号以 "END" 结尾的产品：


```sql
SELECT * FROM products WHERE product_code REGEXP 'END$';
```

#### 5.2.3.查找包含特定字符串的记录

查找订单表中所有描述包含 "紧急" 的订单：


```sql
SELECT * FROM orders WHERE order_description REGEXP '紧急';
```

### 5.3.复杂模式匹配示例

#### 5.3.1.匹配邮箱格式

查找用户表中所有符合邮箱格式的记录：


```sql
SELECT * FROM users WHERE email REGEXP '^[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\\.[A-Za-z]{2,}$';
```

**模式说明**：

- `^[A-Za-z0-9._%+-]+`：邮箱用户名部分，以字母、数字或特定符号开头
- `@`：必须包含 @ 符号
- `[A-Za-z0-9.-]+`：域名部分
- `\\.[A-Za-z]{2,}$`：顶级域名，以点开头，后跟至少 2 个字母

**注意**：在 MySQL 中，点号 `.` 需要转义为 `\\.` 才能匹配字面意思的点。

#### 5.3.2.匹配电话号码格式

查找客户表中所有符合手机号格式的记录（11 位数字，以 1 开头）：


```sql
SELECT * FROM customers WHERE phone_number REGEXP '^1[0-9]{10}$';
```

#### 5.3.3.匹配特定格式的产品编号

查找产品表中所有符合格式 "字母-数字-字母" 的产品编号：


```sql
SELECT * FROM products WHERE product_code REGEXP '^[A-Z]-[0-9]+-[A-Z]$';
```

这个查询会匹配如 `A-123-B`、`X-456-Y` 等格式的编号。

### 5.4.使用 BINARY 进行区分大小写匹配

默认情况下，REGEXP 匹配是不区分大小写的。如果需要区分大小写，可以使用 `BINARY` 关键字：


```sql
SELECT * FROM products WHERE product_name REGEXP BINARY '^Apple';
```

这个查询只会匹配以大写 "A" 开头的 "Apple"，不会匹配 "apple" 或 "APPLE"。

### 5.5.组合多个条件

REGEXP 可以与其他条件组合使用：


```sql
-- 使用 AND
SELECT * FROM products 
WHERE product_code REGEXP '^[A-Z]' 
AND price > 100;

-- 使用 OR
SELECT * FROM products 
WHERE product_name REGEXP '手机|电脑|平板';
```

## 6.RLIKE 操作符的使用

### 6.1.RLIKE 与 REGEXP 的关系

`RLIKE` 是 `REGEXP` 的同义词，两者功能完全相同，可以互换使用。选择使用哪一个主要取决于个人习惯。

### 6.2.RLIKE 基本语法


```sql
SELECT column1, column2, ...
FROM table_name
WHERE column_name RLIKE 'pattern';
```

### 6.3.RLIKE 使用示例

查找产品表中所有名称以数字开头的产品：


```sql
SELECT * FROM products WHERE product_name RLIKE '^[0-9]';
```

查找订单表中所有订单号包含字母和数字组合的记录：


```sql
SELECT * FROM orders WHERE order_number RLIKE '[A-Za-z]+[0-9]+';
```

## 8.正则表达式模式参考表

为了便于查阅，本文整理了 MySQL 中常用的正则表达式模式及其说明：

| 模式       | 描述                                                         | 示例                                                         |
| ---------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| `^`        | 匹配输入字符串的开始位置。如果设置了 RegExp 对象的 Multiline 属性，^ 也匹配 '\n' 或 '\r' 之后的位置。 | `'^abc'` 匹配以 "abc" 开头的字符串                           |
| `$`        | 匹配输入字符串的结束位置。如果设置了 RegExp 对象的 Multiline 属性，$ 也匹配 '\n' 或 '\r' 之前的位置。 | `'xyz$'` 匹配以 "xyz" 结尾的字符串                           |
| `.`        | 匹配除 "\n" 之外的任何单个字符。要匹配包括 '\n' 在内的任何字符，请使用像 '[.\n]' 的模式。 | `'a.c'` 匹配 "abc"、"a1c"、"a_c" 等                          |
| `[...]`    | 字符集合。匹配所包含的任意一个字符。                         | `'[abc]'` 可以匹配 "plain" 中的 'a'                          |
| `[^...]`   | 负值字符集合。匹配未包含的任意字符。                         | `'[^abc]'` 可以匹配 "plain" 中的 'p'、'l'、'i'、'n'          |
| `p1|p2|p3` | 匹配 p1 或 p2 或 p3。                                        | `'z|food'` 能匹配 "z" 或 "food"。`'(z|f)ood'` 则匹配 "zood" 或 "food" |
| 星号       | 匹配前面的子表达式零次或多次。                               | 模式 'zo*' 能匹配 "z" 以及 "zoo"。                           |
| `+`        | 匹配前面的子表达式一次或多次。                               | `'zo+'` 能匹配 "zo" 以及 "zoo"，但不能匹配 "z"。             |
| `{n}`      | n 是一个非负整数。匹配确定的 n 次。                          | `'o{2}'` 不能匹配 "Bob" 中的 'o'，但是能匹配 "food" 中的两个 o |
| `{n,m}`    | m 和 n 均为非负整数，其中 n < = m。最少匹配 n 次且最多匹配 m 次。 | `'o{1,3}'` 将匹配 "fooooood" 中的前三个 o                    |

### 8.1.正则表达式匹配的字符类

除了上述基本模式，MySQL 还支持以下常用的字符类：

| 字符类   | 说明                                     | 示例                                         |
| -------- | ---------------------------------------- | -------------------------------------------- |
| `.`      | 匹配任意单个字符（除换行符）             | `'a.b'` 匹配 "a1b"、"axb" 等                 |
| `^`      | 匹配字符串的开始                         | `'^test'` 匹配以 "test" 开头的字符串         |
| `$`      | 匹配字符串的结束                         | `'test$'` 匹配以 "test" 结尾的字符串         |
| `*`      | 匹配零个或多个前面的元素                 | `'ab*'` 匹配 "a"、"ab"、"abb" 等             |
| `+`      | 匹配一个或多个前面的元素                 | `'ab+'` 匹配 "ab"、"abb" 等，但不匹配 "a"    |
| `?`      | 匹配零个或一个前面的元素                 | `'ab?'` 匹配 "a" 或 "ab"                     |
| `[abc]`  | 匹配字符集中的任意一个字符               | `'[abc]'` 匹配 "a"、"b" 或 "c"               |
| `[^abc]` | 匹配除了字符集中的任意一个字符以外的字符 | `'[^abc]'` 匹配除 "a"、"b"、"c" 外的任意字符 |
| `[a-z]`  | 匹配范围内的任意一个小写字母             | `'[a-z]'` 匹配任意小写字母                   |
| `[0-9]`  | 匹配一个数字字符                         | `'[0-9]'` 匹配任意数字                       |
| `\w`     | 匹配一个字母数字字符（包括下划线）       | 在某些上下文中可用                           |
| `\s`     | 匹配一个空白字符                         | 匹配空格、制表符等                           |

## 9.使用 REGEXP 进行模式匹配

REGEXP 是 MySQL 中用于进行正则表达式匹配的运算符，它允许我们在 WHERE 子句中使用正则表达式来筛选数据。

### 9.1.REGEXP 基本语法

REGEXP 运算符的基本语法如下：


```sql
SELECT column1, column2, ...
FROM table_name
WHERE column_name REGEXP 'pattern';
```

**参数说明**：

- `column1, column2, ...` 是你要选择的列的名称，如果使用 `*` 表示选择所有列
- `table_name` 是你要从中查询数据的表的名称
- `column_name` 是你要进行正则表达式匹配的列的名称
- `'pattern'` 是一个正则表达式模式

### 9.2.基本匹配示例

#### 9.3.2.使用 BINARY 进行区分大小写匹配

## 10.使用 RLIKE 进行模式匹配

RLIKE 是 MySQL 中用于进行正则表达式匹配的运算符，与 REGEXP 功能完全相同。RLIKE 和 REGEXP 可以互换使用，没有区别。选择使用哪一个主要取决于个人习惯或团队规范。

### 10.1.RLIKE 基本语法

以下是使用 RLIKE 进行正则表达式匹配的基本语法：


```sql
SELECT column1, column2, ...
FROM table_name
WHERE column_name RLIKE 'pattern';
```

**参数说明**：

- `column1, column2, ...` 是你要选择的列的名称，如果使用 `*` 表示选择所有列
- `table_name` 是你要从中查询数据的表的名称
- `column_name` 是你要进行正则表达式匹配的列的名称
- `'pattern'` 是一个正则表达式模式

### 10.2.RLIKE 使用示例

#### 10.2.1.查找以数字开头的记录

查找商品表中所有名称以数字开头的商品：


```sql
mysql> SELECT * FROM products 
    -> WHERE product_name RLIKE '^[0-9]';
+------------+--------------+----------+
| product_id | product_name | price    |
+------------+--------------+----------+
| 5          | 2024新款手机  | 2999.00  |
| 8          | 3C数码产品    | 1999.00  |
+------------+--------------+----------+
2 rows in set (0.00 sec)
```

以上 SQL 语句选择产品名称以数字开头的所有产品。

#### 10.2.2.查找包含字母和数字组合的记录

查找订单表中所有订单号包含字母和数字组合的记录：


```sql
mysql> SELECT order_id, order_number, order_date 
    -> FROM orders 
    -> WHERE order_number RLIKE '[A-Za-z]+[0-9]+';
+----------+--------------+---------------------+
| order_id | order_number | order_date          |
+----------+--------------+---------------------+
| 1        | ORD123       | 2024-01-15 10:30:00 |
| 2        | INV456       | 2024-01-16 11:20:00 |
| 3        | PO789        | 2024-01-17 14:15:00 |
+----------+--------------+---------------------+
3 rows in set (0.00 sec)
```

**模式说明**：

- `[A-Za-z]+`：匹配一个或多个字母（大小写均可）
- `[0-9]+`：匹配一个或多个数字

这个查询会匹配所有包含字母后跟数字的订单号。

## 11.实际应用场景

### 11.1.数据验证

正则表达式常用于验证数据的格式是否符合要求。例如，验证邮箱格式、电话号码格式等：


```sql
-- 验证邮箱格式
SELECT * FROM users 
WHERE email REGEXP '^[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\\.[A-Za-z]{2,}$';

-- 验证手机号格式（11位数字，以1开头）
SELECT * FROM customers 
WHERE phone REGEXP '^1[0-9]{10}$';
```

### 11.2.数据清洗

在数据清洗过程中，可以使用正则表达式查找不符合规范的数据：


```sql
-- 查找所有不符合规范的产品编号（应该以字母开头）
SELECT * FROM products 
WHERE product_code REGEXP '^[^A-Za-z]';
```

### 11.3.复杂查询条件

当需要匹配复杂模式时，正则表达式比 LIKE 更灵活：


```sql
-- 查找所有格式为 "字母-数字-字母" 的订单号
SELECT * FROM orders 
WHERE order_number REGEXP '^[A-Z]-[0-9]+-[A-Z]$';
```

## 12.注意事项和最佳实践

### 12.1.性能考虑

- **索引使用**：REGEXP 和 RLIKE 通常无法使用索引，在大数据量查询时可能影响性能
- **简单查询优先**：对于简单的模式匹配，优先考虑使用 LIKE，性能更好
- **复杂模式使用正则**：只有在需要复杂模式匹配时才使用正则表达式

### 12.2.转义字符

在正则表达式中，某些字符具有特殊含义。如果要匹配这些字符的字面意思，需要进行转义：


```sql
-- 匹配包含点号的字符串
SELECT * FROM products 
WHERE product_code REGEXP '\\.';

-- 匹配包含星号的字符串
SELECT * FROM products 
WHERE description REGEXP '\\*';
```

### 12.3.大小写敏感性

默认情况下，REGEXP 和 RLIKE 的匹配是不区分大小写的。如果需要区分大小写，使用 `BINARY` 关键字：


```sql
SELECT * FROM products 
WHERE product_name REGEXP BINARY '^Apple';
```

### 12.4.模式复杂度

- 保持模式尽可能简单，复杂的正则表达式可能难以理解和维护
- 对于非常复杂的模式，考虑在应用层进行验证，而不是在数据库层
