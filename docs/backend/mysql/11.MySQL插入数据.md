## 1.引言

在创建数据表后，下一步就是向表中插入数据。数据插入是数据库操作的基础，掌握插入数据的方法对于使用 MySQL 至关重要。

本文将从插入数据的基础概念开始，逐步介绍如何使用 `INSERT INTO` 语句向 MySQL 数据表插入数据，包括基本语法、单行插入、多行插入，以及如何通过命令行和编程脚本来执行插入操作。

## 2.插入数据基础概念

### 2.1.什么是插入数据？

插入数据是指向已存在的数据表中添加新记录（行）的操作。每执行一次插入操作，就会在表中增加一行新的数据。

**类比理解**：

- 数据表就像是一个 Excel 表格
- 插入数据就像是在表格中添加新的一行
- 每一行数据包含多个字段的值，就像表格中每一行有多个单元格

### 2.2.插入数据的前提条件

在插入数据之前，需要满足以下条件：

- **数据表已存在**：必须先创建好数据表，才能向其中插入数据
- **字段匹配**：插入的数据必须与表的字段定义相匹配
- **数据类型正确**：插入的值必须符合对应字段的数据类型要求
- **约束条件满足**：插入的数据必须满足表的约束条件（如主键唯一性、非空约束等）

### 2.3.插入数据的方式

MySQL 提供了多种插入数据的方式：

- **指定列名插入**：明确指定要插入的列名和对应的值，更灵活、更安全
- **省略列名插入**：不指定列名，按表的字段顺序插入所有列的值
- **批量插入**：一次插入多行数据，提高效率

## 3.INSERT INTO 语句基本语法

### 3.1.基本语法格式

向 MySQL 数据表插入数据使用 `INSERT INTO` 语句，基本语法如下：


```sql
INSERT INTO table_name (column1, column2, column3, ...)
VALUES (value1, value2, value3, ...);
```

**参数说明**：

- `table_name`：要插入数据的表名
- `column1, column2, column3, ...`：表中的列名，指定要插入数据的列
- `value1, value2, value3, ...`：要插入的具体数值，与列名一一对应

### 3.2.数据类型注意事项

插入数据时，需要注意数据类型：

- 字符串类型

  ：必须使用单引号

   

  ```
  '
  ```

   

  或双引号

   

  ```
  "
  ```

   

  括起来


  ```sql
  '文本内容' "文本内容"
  ```

- 数值类型

  ：直接写入，不需要引号


  ```sql
  100 3.14
  ```

- 日期类型

  ：使用单引号括起来，格式通常为

   

  ```
  'YYYY-MM-DD'
  ```


  ```sql
  '2024-01-15'
  ```

- **布尔类型**：使用 `TRUE` 或 `FALSE`，或者使用 `1` 和 `0`

### 3.3.基本插入示例

假设我们有一个 `products` 表，包含以下字段：

- `product_id`：产品ID（自增主键）
- `product_name`：产品名称
- `price`：价格
- `stock`：库存数量
- `created_at`：创建时间

**示例：插入单行数据**


```sql
INSERT INTO products (product_name, price, stock, created_at)
VALUES ('笔记本电脑', 5999.00, 50, '2024-01-15');
```

**字段说明**：

- `product_name`：产品名称，字符串类型
- `price`：价格，数值类型
- `stock`：库存数量，整数类型
- `created_at`：创建时间，日期类型

**注意**：这里没有插入 `product_id` 的值，因为该字段在创建表时设置为 `AUTO_INCREMENT`（自动增长），MySQL 会自动为其生成一个唯一的值。

## 4.插入数据的几种方式

### 4.1.指定列名插入（推荐）

指定列名插入是最常用和推荐的方式，因为它具有以下优点：

- **灵活性高**：可以选择性地插入部分字段
- **可读性强**：明确知道哪些字段被插入了什么值
- **安全性好**：即使表结构发生变化，只要指定的列存在，语句仍然有效

**语法格式**：


```sql
INSERT INTO table_name (column1, column2, column3)
VALUES (value1, value2, value3);
```

**示例**：


```sql
INSERT INTO products (product_name, price, stock)
VALUES ('智能手机', 3999.00, 100);
```

在这个例子中，我们只插入了 `product_name`、`price` 和 `stock` 三个字段，`product_id` 会自动生成，`created_at` 如果设置了默认值或允许为 NULL，则使用默认值或 NULL。

### 4.2.省略列名插入

如果要将数据插入到表的所有列中，可以省略列名，但必须按照表定义时的字段顺序提供所有字段的值。

**语法格式**：


```sql
INSERT INTO table_name
VALUES (value1, value2, value3, ...);
```

**示例**：


```sql
INSERT INTO products
VALUES (NULL, '平板电脑', 2999.00, 30, '2024-01-20');
```

**注意事项**：

- 必须提供所有字段的值，不能遗漏
- 值的顺序必须与表定义时的字段顺序完全一致
- 对于自增字段，可以使用 `NULL` 作为占位符，MySQL 会自动生成值
- 如果字段允许为 NULL 且没有默认值，也可以使用 `NULL`

**使用建议**：

- 不推荐使用这种方式，因为表结构变化时容易出错
- 只有在确定要插入所有字段且顺序不会改变时才使用

### 4.3.批量插入多行数据

如果需要一次性插入多行数据，可以在 `VALUES` 子句中指定多组数值，每组数值用逗号分隔。

**语法格式**：


```sql
INSERT INTO table_name (column1, column2, column3)
VALUES
    (value1, value2, value3),
    (value4, value5, value6),
    (value7, value8, value9);
```

**示例**：


```sql
INSERT INTO products (product_name, price, stock, created_at)
VALUES
    ('无线耳机', 299.00, 200, '2024-01-15'),
    ('机械键盘', 599.00, 150, '2024-01-16'),
    ('游戏鼠标', 199.00, 300, '2024-01-17');
```

这个语句会在 `products` 表中一次性插入三行数据。

**批量插入的优点**：

- **效率高**：一次执行插入多行，比多次执行单行插入更快
- **事务安全**：所有行要么全部插入成功，要么全部失败，保证数据一致性
- **代码简洁**：减少重复的 SQL 语句

## 5.使用 MySQL 命令提示窗口插入数据

### 5.1.连接到 MySQL

首先需要连接到 MySQL 服务器。在命令行窗口中执行：


```bash
mysql -u root -p
```

**命令说明**：

- `-u root`：指定用户名为 root
- `-p`：提示输入密码

执行后会提示输入密码，输入正确的密码后按回车键。

### 5.2.选择数据库

连接成功后，需要选择要使用的数据库：


```sql
USE your_database_name;
```

**示例**：


```sql
USE shop_db;
```

执行后会显示 `Database changed`，表示已成功切换到指定数据库。

### 5.3.插入数据示例

假设我们有一个 `employees` 表，包含以下字段：

- `emp_id`：员工ID（自增主键）
- `emp_name`：员工姓名
- `department`：部门
- `salary`：薪资
- `hire_date`：入职日期

**示例一：插入单行数据**


```sql
mysql> INSERT INTO employees (emp_name, department, salary, hire_date)
    -> VALUES ('张三', '技术部', 12000.00, '2023-06-01');
Query OK, 1 row affected (0.02 sec)
```

**示例二：插入多行数据**


```sql
mysql> INSERT INTO employees (emp_name, department, salary, hire_date)
    -> VALUES
    -> ('李四', '销售部', 8000.00, '2023-07-15'),
    -> ('王五', '人事部', 7500.00, '2023-08-20'),
    -> ('赵六', '技术部', 15000.00, '2023-09-10');
Query OK, 3 rows affected (0.01 sec)
```

**示例三：使用函数插入数据**

MySQL 提供了许多有用的函数，可以在插入数据时使用：


```sql
mysql> INSERT INTO employees (emp_name, department, salary, hire_date)
    -> VALUES ('孙七', '财务部', 9000.00, NOW());
Query OK, 1 row affected (0.01 sec)
```

这里使用了 `NOW()` 函数，它会自动插入当前的日期和时间。

**关于命令提示符的说明**：

- `mysql>` 是 MySQL 的命令提示符，表示可以输入 SQL 语句
- `->` 是续行提示符，表示上一行命令未结束，可以继续输入
- 如果一条 SQL 语句太长，可以按回车键换行继续输入
- SQL 语句以分号 `;` 结束，输入分号后按回车键执行

## 6.插入数据时的常见问题

### 6.1.数据类型不匹配

**问题**：插入的数据类型与字段定义不匹配

**示例错误**：


```sql
-- 错误：price 字段是数值类型，不能用引号
INSERT INTO products (product_name, price) VALUES ('商品', '5999');
```

**解决方法**：确保数据类型匹配


```sql
-- 正确：数值类型不需要引号
INSERT INTO products (product_name, price) VALUES ('商品', 5999);
```

### 6.2.字段数量不匹配

**问题**：插入的值数量与指定的列数量不一致

**示例错误**：


```sql
-- 错误：指定了 3 个列，但只提供了 2 个值
INSERT INTO products (product_name, price, stock) VALUES ('商品', 5999);
```

**解决方法**：确保值的数量与列的数量一致


```sql
-- 正确：值的数量与列的数量一致
INSERT INTO products (product_name, price, stock) VALUES ('商品', 5999, 100);
```

### 6.3.违反约束条件

**问题**：插入的数据违反了表的约束条件（如主键重复、非空约束等）

**示例错误**：


```sql
-- 错误：如果 product_id 是主键且已存在值为 1 的记录
INSERT INTO products (product_id, product_name) VALUES (1, '新商品');
```

**解决方法**：

- 对于自增主键，不要手动指定值，让 MySQL 自动生成
- 检查唯一性约束，确保不插入重复值
- 检查非空约束，确保必填字段有值

### 6.4.字符串引号问题

**问题**：字符串值没有使用引号，或引号使用不当

**示例错误**：


```sql
-- 错误：字符串没有使用引号
INSERT INTO products (product_name) VALUES (笔记本电脑);
```

**解决方法**：字符串必须使用单引号或双引号


```sql
-- 正确：字符串使用引号
INSERT INTO products (product_name) VALUES ('笔记本电脑');
-- 或使用双引号
INSERT INTO products (product_name) VALUES ("笔记本电脑");
```

### 6.5.日期格式问题

**问题**：日期格式不正确，导致插入失败

**示例错误**：


```sql
-- 错误：日期格式不正确
INSERT INTO products (product_name, created_at) VALUES ('商品', '2024/01/15');
```

**解决方法**：使用标准的日期格式 `'YYYY-MM-DD'`


```sql
-- 正确：使用标准日期格式
INSERT INTO products (product_name, created_at) VALUES ('商品', '2024-01-15');
```

### 6.6.自增字段处理

**问题**：手动指定自增字段的值可能导致冲突

**示例错误**：


```sql
-- 如果 product_id 是自增字段，手动指定可能导致问题
INSERT INTO products (product_id, product_name) VALUES (100, '商品');
```

**解决方法**：对于自增字段，使用 `NULL` 或直接省略


```sql
-- 方法一：使用 NULL
INSERT INTO products (product_id, product_name) VALUES (NULL, '商品');

-- 方法二：直接省略自增字段（推荐）
INSERT INTO products (product_name) VALUES ('商品');
```

## 7.插入数据的高级技巧

### 7.1.使用 INSERT ... SELECT 语句

如果需要从另一个表复制数据到当前表，可以使用 `INSERT ... SELECT` 语句：

**语法格式**：


```sql
INSERT INTO table_name1 (column1, column2, ...)
SELECT column1, column2, ...
FROM table_name2
WHERE condition;
```

**示例**：从 `products` 表复制数据到 `products_backup` 表


```sql
INSERT INTO products_backup (product_name, price, stock)
SELECT product_name, price, stock
FROM products
WHERE price > 1000;
```

### 7.2.使用 INSERT ... ON DUPLICATE KEY UPDATE

当插入的数据与现有记录的主键或唯一键冲突时，可以使用 `ON DUPLICATE KEY UPDATE` 来更新现有记录：

**语法格式**：


```sql
INSERT INTO table_name (column1, column2, ...)
VALUES (value1, value2, ...)
ON DUPLICATE KEY UPDATE
    column1 = value1,
    column2 = value2;
```

**示例**：


```sql
INSERT INTO products (product_id, product_name, price, stock)
VALUES (1, '更新后的商品名', 6999.00, 60)
ON DUPLICATE KEY UPDATE
    product_name = '更新后的商品名',
    price = 6999.00,
    stock = 60;
```

如果 `product_id = 1` 的记录已存在，则更新该记录；如果不存在，则插入新记录。

### 7.3.使用 INSERT IGNORE

如果希望插入数据时忽略错误（如主键冲突），可以使用 `INSERT IGNORE`：

**语法格式**：


```sql
INSERT IGNORE INTO table_name (column1, column2, ...)
VALUES (value1, value2, ...);
```

**示例**：


```sql
INSERT IGNORE INTO products (product_id, product_name, price)
VALUES (1, '商品A', 5999.00);
```

如果 `product_id = 1` 已存在，该语句不会报错，也不会插入数据；如果不存在，则正常插入。

### 7.4.使用 REPLACE INTO

`REPLACE INTO` 语句会在插入数据时，如果遇到主键或唯一键冲突，会先删除旧记录，再插入新记录：

**语法格式**：


```sql
REPLACE INTO table_name (column1, column2, ...)
VALUES (value1, value2, ...);
```

**示例**：


```sql
REPLACE INTO products (product_id, product_name, price, stock)
VALUES (1, '替换后的商品', 7999.00, 40);
```

**注意**：`REPLACE INTO` 会删除旧记录，可能导致数据丢失，使用时要谨慎。

### 7.5.插入时使用函数和表达式

在插入数据时，可以使用 MySQL 函数和表达式来计算值：

**示例一：使用 NOW() 函数插入当前时间**


```sql
INSERT INTO products (product_name, price, stock, created_at)
VALUES ('新商品', 2999.00, 100, NOW());
```

**示例二：使用表达式计算值**


```sql
INSERT INTO products (product_name, price, stock, discount_price)
VALUES ('促销商品', 1000.00, 50, 1000.00 * 0.8);
```

这里 `discount_price` 会被计算为 `800.00`（原价的 80%）。

**示例三：使用字符串函数**


```sql
INSERT INTO products (product_name, price)
VALUES (UPPER('新商品'), 1999.00);
```

这里 `product_name` 会被插入为 `'新商品'` 的大写形式。

## 8.实践建议

### 8.1.优先使用指定列名的方式

建议始终使用指定列名的方式插入数据，因为：

- 代码更清晰，易于理解和维护
- 即使表结构发生变化，只要指定的列存在，语句仍然有效
- 可以选择性地插入部分字段，更灵活

### 8.2.批量插入提高效率

当需要插入大量数据时，使用批量插入比多次单行插入效率更高：


```sql
-- 推荐：批量插入
INSERT INTO products (product_name, price, stock)
VALUES
    ('商品1', 100, 50),
    ('商品2', 200, 60),
    ('商品3', 300, 70);

-- 不推荐：多次单行插入
INSERT INTO products (product_name, price, stock) VALUES ('商品1', 100, 50);
INSERT INTO products (product_name, price, stock) VALUES ('商品2', 200, 60);
INSERT INTO products (product_name, price, stock) VALUES ('商品3', 300, 70);
```

### 8.3.使用事务保证数据一致性

在插入多条相关数据时，使用事务可以保证数据的一致性：


```sql
START TRANSACTION;

INSERT INTO orders (order_date, customer_id) VALUES ('2024-01-20', 1);
INSERT INTO order_items (order_id, product_id, quantity) VALUES (LAST_INSERT_ID(), 1, 2);
INSERT INTO order_items (order_id, product_id, quantity) VALUES (LAST_INSERT_ID(), 2, 1);

COMMIT;
```

如果任何一条插入语句失败，可以使用 `ROLLBACK` 回滚所有操作。

### 8.4.处理字符编码问题

插入中文数据时，确保：

- 数据库和表的字符集设置为 `utf8mb4`
- 连接时指定正确的字符集
- 在编程语言中设置正确的编码

**检查字符集的 SQL**：


```sql
-- 查看数据库字符集
SHOW CREATE DATABASE your_database;

-- 查看表字符集
SHOW CREATE TABLE your_table;
```

### 8.5.验证插入结果

插入数据后，建议立即查询验证数据是否正确插入：


```sql
-- 插入数据
INSERT INTO products (product_name, price, stock) VALUES ('测试商品', 999.00, 10);

-- 验证插入结果
SELECT * FROM products WHERE product_name = '测试商品';
```

### 8.6.使用参数化查询防止 SQL 注入

在编程语言中插入数据时，使用参数化查询（预处理语句）可以防止 SQL 注入攻击：


```python
# 推荐：使用参数化查询
sql = "INSERT INTO products (product_name, price) VALUES (%s, %s)"
values = ('商品名', 999.00)
cursor.execute(sql, values)

# 不推荐：直接拼接 SQL（存在 SQL 注入风险）
sql = f"INSERT INTO products (product_name, price) VALUES ('{product_name}', {price})"
cursor.execute(sql)
```
