## 1.引言

临时表是 MySQL 中一种特殊的数据表，用于存储临时数据。与普通表不同，临时表只在当前数据库连接中可见，当连接关闭时会自动删除。

本文将从临时表的基础概念开始，逐步介绍如何创建、使用和管理临时表，包括创建临时表、插入数据、查询数据、修改表结构以及删除临时表等操作。通过实际示例，你将学会如何在实际场景中运用临时表来处理临时数据。

## 2.临时表基础概念

### 2.1.什么是临时表？

临时表（Temporary Table）是 MySQL 提供的一种特殊类型的表，用于存储临时数据。临时表只在创建它的数据库连接（会话）中可见，其他连接无法访问。

**临时表的特点**：

- **会话隔离**：每个数据库连接都有自己独立的临时表空间
- **自动清理**：当连接关闭时，临时表会自动被删除
- **性能优化**：临时表通常存储在内存中，访问速度快
- **不影响永久数据**：临时表的数据不会影响数据库中的永久表

### 2.2.临时表与普通表的区别

理解临时表和普通表的区别很重要：

| 特性     | 临时表                 | 普通表               |
| -------- | ---------------------- | -------------------- |
| 可见性   | 仅当前连接可见         | 所有连接可见         |
| 生命周期 | 连接关闭时自动删除     | 永久存在，需手动删除 |
| 命名冲突 | 可以与普通表同名       | 不能与已存在的表同名 |
| 用途     | 存储临时数据、中间结果 | 存储永久数据         |

**类比理解**：

- 普通表就像是一个永久保存的文档
- 临时表就像是一个草稿纸，用完就可以扔掉
- 不同的人（不同的连接）有各自的草稿纸，互不干扰

### 2.3.临时表的应用场景

临时表适用于以下场景：

- **复杂查询的中间结果**：在进行多步骤查询时，将中间结果存储在临时表中
- **数据分析和报表生成**：临时存储计算过程中的数据
- **批量数据处理**：处理大量数据时，使用临时表暂存处理结果
- **测试和调试**：在开发和测试过程中，使用临时表进行数据测试

## 3.创建临时表

### 3.1.基本语法

创建临时表使用 `CREATE TEMPORARY TABLE` 语句，基本语法如下：


```sql
CREATE TEMPORARY TABLE temp_table_name (
    column1 datatype,
    column2 datatype,
    ...
);
```

**参数说明**：

- `TEMPORARY`：关键字，表示创建的是临时表
- `temp_table_name`：临时表的名称
- `column1`, `column2`：表中的列名（字段名）
- `datatype`：每个列的数据类型

### 3.2.从查询结果创建临时表

除了直接定义表结构，还可以基于查询结果创建临时表：


```sql
CREATE TEMPORARY TABLE temp_table_name AS
SELECT column1, column2, ...
FROM source_table
WHERE condition;
```

这种方式可以直接将查询结果存储到临时表中，非常方便。

### 3.3.创建临时表示例

下面是一个创建学生成绩临时表的示例：


```sql
-- 创建一个临时表，用于存储学生成绩信息
CREATE TEMPORARY TABLE student_scores (
    -- 学生编号，整数类型，自动递增，作为主键
    student_id INT AUTO_INCREMENT PRIMARY KEY,
    -- 学生姓名，变长字符串，最大长度50，不允许为空
    student_name VARCHAR(50) NOT NULL,
    -- 科目名称，变长字符串，最大长度30
    subject VARCHAR(30),
    -- 成绩，精确小数类型，总共5位，小数点后2位
    score DECIMAL(5, 2),
    -- 考试日期，日期类型
    exam_date DATE
);
```

**字段说明**：

- `student_id`：学生编号，自动递增的主键
- `student_name`：学生姓名，不允许为空
- `subject`：科目名称
- `score`：成绩，使用小数类型存储
- `exam_date`：考试日期

### 3.4.从现有表创建临时表

基于现有表的数据创建临时表：


```sql
-- 从学生表中筛选出2024年的学生记录，创建临时表
CREATE TEMPORARY TABLE recent_students AS
SELECT student_id, student_name, enrollment_date
FROM students
WHERE enrollment_date >= '2024-01-01';
```

这个示例会创建一个临时表，包含所有在 2024 年及之后注册的学生信息。

## 4.操作临时表

### 4.1.插入数据到临时表

向临时表插入数据的方法与普通表完全相同，使用 `INSERT INTO` 语句：


```sql
INSERT INTO temp_table_name (column1, column2, ...)
VALUES (value1, value2, ...);
```

**示例：插入单条数据**


```sql
-- 向学生成绩临时表插入一条记录
INSERT INTO student_scores (student_name, subject, score, exam_date)
VALUES ('张三', '数学', 95.50, '2024-03-15');
```

**示例：插入多条数据**


```sql
-- 向临时表插入多条学生成绩记录
INSERT INTO student_scores (student_name, subject, score, exam_date)
VALUES 
    ('李四', '语文', 88.00, '2024-03-15'),
    ('王五', '英语', 92.50, '2024-03-16'),
    ('赵六', '数学', 87.75, '2024-03-15');
```

### 4.2.查询临时表

查询临时表使用标准的 `SELECT` 语句：


```sql
SELECT * FROM temp_table_name;
```

**示例：查询所有数据**


```sql
-- 查询临时表中的所有学生成绩记录
SELECT * FROM student_scores;
```

**示例：条件查询**


```sql
-- 查询数学成绩大于90分的学生
SELECT student_name, score
FROM student_scores
WHERE subject = '数学' AND score > 90;
```

**示例：统计查询**


```sql
-- 统计每个科目的平均分
SELECT subject, AVG(score) AS average_score
FROM student_scores
GROUP BY subject;
```

### 4.3.修改临时表结构

临时表的结构可以使用 `ALTER TABLE` 命令进行修改，操作方式与普通表相同：

**添加新列**


```sql
-- 在临时表中添加一个新列，用于存储备注信息
ALTER TABLE student_scores
ADD COLUMN remarks VARCHAR(200);
```

**修改列类型**


```sql
-- 修改成绩列的数据类型，增加精度
ALTER TABLE student_scores
MODIFY COLUMN score DECIMAL(6, 2);
```

**删除列**


```sql
-- 删除临时表中的备注列
ALTER TABLE student_scores
DROP COLUMN remarks;
```

### 4.4.更新临时表数据

使用 `UPDATE` 语句可以修改临时表中的数据：


```sql
-- 将张三的数学成绩更新为98分
UPDATE student_scores
SET score = 98.00
WHERE student_name = '张三' AND subject = '数学';
```

### 4.5.删除临时表数据

使用 `DELETE` 语句可以删除临时表中的数据：


```sql
-- 删除成绩低于60分的记录
DELETE FROM student_scores
WHERE score < 60;
```

## 5.删除临时表

### 5.1.自动删除

临时表在以下情况下会自动被删除：

- **连接关闭时**：当数据库连接断开时，该连接创建的所有临时表都会自动删除
- **会话结束时**：在命令行中退出 MySQL 客户端时，临时表会被删除
- **脚本执行完成**：如果通过脚本（如 PHP、Python）创建临时表，脚本执行完成后临时表会被删除

### 5.2.手动删除临时表

虽然临时表会自动删除，但也可以使用 `DROP TABLE` 命令手动删除：


```sql
DROP TEMPORARY TABLE IF EXISTS temp_table_name;
```

**参数说明**：

- `TEMPORARY`：关键字，明确表示删除的是临时表（可选，但建议使用）
- `IF EXISTS`：如果表不存在也不会报错，避免错误信息

**示例：删除临时表**


```sql
-- 删除学生成绩临时表
DROP TEMPORARY TABLE IF EXISTS student_scores;
```

**注意**：删除临时表后，表中的所有数据都会丢失，且无法恢复。

## 6.实际应用示例

### 6.1.示例一：数据分析场景

假设我们需要分析最近一个月的销售数据，可以使用临时表存储筛选后的数据：


```sql
-- 创建临时表，存储最近一个月的订单数据
CREATE TEMPORARY TABLE monthly_orders AS
SELECT order_id, customer_name, product_name, order_amount, order_date
FROM orders
WHERE order_date >= DATE_SUB(CURDATE(), INTERVAL 1 MONTH);

-- 查询临时表中的数据，按客户分组统计总金额
SELECT customer_name, SUM(order_amount) AS total_amount
FROM monthly_orders
GROUP BY customer_name
ORDER BY total_amount DESC;

-- 使用完毕后删除临时表
DROP TEMPORARY TABLE IF EXISTS monthly_orders;
```

### 6.2.示例二：复杂查询优化

在进行复杂的多表关联查询时，可以先将中间结果存储在临时表中：


```sql
-- 第一步：创建临时表，存储所有活跃用户的信息
CREATE TEMPORARY TABLE active_users AS
SELECT user_id, username, email, last_login_date
FROM users
WHERE status = 'active' AND last_login_date >= '2024-01-01';

-- 第二步：基于临时表进行进一步查询，统计每个用户的订单数
SELECT 
    au.username,
    au.email,
    COUNT(o.order_id) AS order_count
FROM active_users au
LEFT JOIN orders o ON au.user_id = o.user_id
GROUP BY au.user_id, au.username, au.email;

-- 清理临时表
DROP TEMPORARY TABLE IF EXISTS active_users;
```

下面展示一个完整的临时表使用流程：


```sql

CREATE TEMPORARY TABLE product_inventory (

-- 步骤2：向临时表插入数据

```

## 7.临时表的注意事项

### 7.1.命名冲突

临时表可以与普通表使用相同的名称，在同一个连接中，临时表会优先于普通表：


```sql
-- 假设数据库中已经存在一个名为 users 的普通表
-- 可以创建一个同名的临时表
CREATE TEMPORARY TABLE users (
    id INT,
    name VARCHAR(50)
);

-- 此时查询 users 表，会优先访问临时表
SELECT * FROM users;  -- 访问的是临时表，而不是普通表
```

**重要提示**：虽然可以同名，但为了避免混淆，建议临时表使用不同的命名规范，比如添加 `temp_` 前缀。

### 7.2.临时表的可见性

临时表只在创建它的连接中可见，其他连接无法访问：


```sql
-- 连接A中创建临时表
CREATE TEMPORARY TABLE temp_data (
    id INT,
    value VARCHAR(50)
);

-- 在连接A中可以正常访问
SELECT * FROM temp_data;

-- 但在连接B中无法看到这个临时表
-- 如果连接B执行 SHOW TABLES，不会显示 temp_data
```

### 7.3.使用 SHOW TABLES 查看临时表

使用 `SHOW TABLES` 命令时，默认情况下不会显示临时表。如果需要查看临时表，可以使用以下方式：


```sql
-- 查看所有表（包括临时表）
SHOW FULL TABLES;

-- 或者查看特定数据库的所有表
SHOW TABLES FROM database_name;
```

**注意**：即使使用 `SHOW FULL TABLES`，临时表也只会在创建它的连接中显示。

### 7.4.临时表的存储位置

临时表的数据存储位置取决于表的大小和 MySQL 配置：

- **小表**：通常存储在内存中（使用 MEMORY 存储引擎），访问速度快
- **大表**：当数据量超过 `tmp_table_size` 或 `max_heap_table_size` 时，会转换为磁盘存储
- **存储位置**：磁盘临时表存储在 MySQL 的临时目录中（由 `tmpdir` 变量指定）

可以通过以下命令查看临时表使用的存储引擎：


```sql
SHOW CREATE TABLE temp_table_name;
```

## 8.完整实践示例

### 8.1.示例一：销售数据统计

假设我们需要统计本季度的产品销售情况，可以使用临时表来存储和处理数据：


```sql
-- 创建临时表，存储本季度的销售记录
CREATE TEMPORARY TABLE quarterly_sales AS
SELECT 
    product_id,
    product_name,
    sale_date,
    sale_amount,
    quantity
FROM sales
WHERE sale_date >= '2024-01-01' AND sale_date < '2024-04-01';

-- 查询临时表中的数据
SELECT * FROM quarterly_sales;

-- 按产品统计总销售额和总数量
SELECT 
    product_name,
    SUM(sale_amount) AS total_revenue,
    SUM(quantity) AS total_quantity
FROM quarterly_sales
GROUP BY product_id, product_name
ORDER BY total_revenue DESC;

-- 向临时表插入新的测试数据
INSERT INTO quarterly_sales (product_id, product_name, sale_date, sale_amount, quantity)
VALUES (101, '测试商品', '2024-02-15', 500.00, 5);

-- 再次查询验证数据
SELECT * FROM quarterly_sales WHERE product_name = '测试商品';

-- 使用完毕后删除临时表
DROP TEMPORARY TABLE IF EXISTS quarterly_sales;
```

### 8.2.示例二：员工绩效分析

下面展示如何使用临时表进行员工绩效分析：


```sql
-- 创建临时表，定义员工绩效统计表结构
CREATE TEMPORARY TABLE employee_performance (
    emp_id INT AUTO_INCREMENT PRIMARY KEY,
    employee_name VARCHAR(50) NOT NULL,
    department VARCHAR(30),
    project_count INT DEFAULT 0,
    total_score DECIMAL(8, 2) DEFAULT 0.00,
    average_score DECIMAL(5, 2) DEFAULT 0.00
);

-- 从员工表和项目表中提取数据并插入临时表
INSERT INTO employee_performance (employee_name, department, project_count, total_score, average_score)
SELECT 
    e.name,
    e.department,
    COUNT(p.project_id) AS project_count,
    SUM(p.score) AS total_score,
    AVG(p.score) AS average_score
FROM employees e
LEFT JOIN projects p ON e.emp_id = p.employee_id
WHERE p.completion_date >= '2024-01-01'
GROUP BY e.emp_id, e.name, e.department;

-- 查询所有员工绩效数据
SELECT * FROM employee_performance;

-- 查询平均分高于85分的员工
SELECT employee_name, department, average_score
FROM employee_performance
WHERE average_score > 85.00
ORDER BY average_score DESC;

-- 修改临时表结构，添加排名列
ALTER TABLE employee_performance
ADD COLUMN performance_rank INT;

-- 更新排名数据（这里简化处理，实际可以使用窗口函数）
UPDATE employee_performance
SET performance_rank = (
    SELECT COUNT(*) + 1
    FROM employee_performance ep2
    WHERE ep2.average_score > employee_performance.average_score
);

-- 查看最终结果
SELECT 
    employee_name,
    department,
    project_count,
    average_score,
    performance_rank
FROM employee_performance
ORDER BY performance_rank;

-- 删除临时表
DROP TEMPORARY TABLE IF EXISTS employee_performance;
```

### 8.4.示例四：从查询结果创建临时表

在实际应用中，经常需要基于查询结果创建临时表，这样可以方便地对数据进行进一步处理：


```sql
-- 从订单表中筛选出2024年第一季度的订单，创建临时表
CREATE TEMPORARY TABLE q1_orders AS
SELECT 
    order_id,
    customer_id,
    order_date,
    total_amount,
    order_status
FROM orders
WHERE order_date >= '2024-01-01' AND order_date < '2024-04-01';

-- 查询临时表中的数据总数
SELECT COUNT(*) AS total_orders FROM q1_orders;

-- 按客户统计订单金额
SELECT 
    customer_id,
    COUNT(*) AS order_count,
    SUM(total_amount) AS total_spent
FROM q1_orders
WHERE order_status = 'completed'
GROUP BY customer_id
ORDER BY total_spent DESC;

-- 向临时表插入一条测试数据
INSERT INTO q1_orders (order_id, customer_id, order_date, total_amount, order_status)
VALUES (9999, 100, '2024-02-20', 888.00, 'pending');

-- 查询刚插入的数据
SELECT * FROM q1_orders WHERE order_id = 9999;

-- 更新订单状态
UPDATE q1_orders
SET order_status = 'completed'
WHERE order_id = 9999;

-- 删除测试数据
DELETE FROM q1_orders WHERE order_id = 9999;

-- 清理临时表
DROP TEMPORARY TABLE IF EXISTS q1_orders;
```

### 8.5.示例五：临时表在复杂查询中的应用

当需要进行多步骤的复杂查询时，临时表可以帮助简化查询逻辑：


```sql
-- 第一步：创建临时表，存储所有高价值客户的信息
CREATE TEMPORARY TABLE vip_customers AS
SELECT 
    c.customer_id,
    c.customer_name,
    c.email,
    SUM(o.order_amount) AS lifetime_value
FROM customers c
INNER JOIN orders o ON c.customer_id = o.customer_id
WHERE o.order_date >= '2023-01-01'
GROUP BY c.customer_id, c.customer_name, c.email
HAVING SUM(o.order_amount) > 10000;

-- 第二步：基于临时表，查询这些客户的最近订单
SELECT 
    vc.customer_name,
    vc.lifetime_value,
    o.order_id,
    o.order_date,
    o.order_amount
FROM vip_customers vc
LEFT JOIN orders o ON vc.customer_id = o.customer_id
WHERE o.order_date >= DATE_SUB(CURDATE(), INTERVAL 30 DAY)
ORDER BY vc.lifetime_value DESC, o.order_date DESC;

-- 第三步：统计每个VIP客户的订单数量
SELECT 
    customer_name,
    lifetime_value,
    COUNT(o.order_id) AS recent_order_count
FROM vip_customers vc
LEFT JOIN orders o ON vc.customer_id = o.customer_id 
    AND o.order_date >= DATE_SUB(CURDATE(), INTERVAL 30 DAY)
GROUP BY vc.customer_id, customer_name, lifetime_value;

-- 清理临时表
DROP TEMPORARY TABLE IF EXISTS vip_customers;
```

## 9.临时表的特殊行为

### 9.1.临时表与普通表的优先级

当临时表与普通表同名时，在当前连接中，临时表会优先被访问：


```sql
-- 假设数据库中已存在一个名为 products 的普通表
-- 创建一个同名的临时表
CREATE TEMPORARY TABLE products (
    id INT,
    name VARCHAR(100),
    price DECIMAL(10, 2)
);

-- 此时查询 products，会访问临时表而不是普通表
SELECT * FROM products;  -- 返回的是临时表的数据

-- 删除临时表后，再次查询会访问普通表
DROP TEMPORARY TABLE products;
SELECT * FROM products;  -- 现在返回的是普通表的数据
```

**重要提示**：虽然 MySQL 允许临时表与普通表同名，但为了避免混淆和潜在的错误，建议临时表使用不同的命名规范，例如添加 `tmp_` 或 `temp_` 前缀。

### 9.2.临时表的可见性验证

临时表只在创建它的连接中可见，可以通过以下方式验证：


```sql
-- 在连接A中执行
CREATE TEMPORARY TABLE session_data (
    id INT PRIMARY KEY,
    data_value VARCHAR(100)
);

INSERT INTO session_data VALUES (1, '连接A的数据');

-- 在连接A中可以正常查询
SELECT * FROM session_data;  -- 可以正常返回数据

-- 在连接B中执行 SHOW TABLES
SHOW TABLES;  -- 不会显示 session_data 临时表

-- 在连接B中尝试查询临时表会报错
SELECT * FROM session_data;  -- ERROR 1146: Table doesn't exist
```

### 9.3.查看临时表的方法

使用 `SHOW TABLES` 命令时，默认情况下不会显示临时表。如果需要查看当前连接中创建的临时表，可以使用以下方法：


```sql
-- 方法一：使用 SHOW FULL TABLES 命令
SHOW FULL TABLES;

-- 方法二：查询 INFORMATION_SCHEMA
SELECT TABLE_NAME, TABLE_TYPE
FROM INFORMATION_SCHEMA.TABLES
WHERE TABLE_SCHEMA = DATABASE()
AND TABLE_TYPE = 'TEMPORARY';

-- 方法三：直接尝试查询（如果表存在则不会报错）
SELECT COUNT(*) FROM temp_table_name;
```

**注意**：即使使用这些方法，临时表也只会在创建它的连接中显示，其他连接无法看到。

### 9.4.临时表的存储机制

MySQL 会根据临时表的大小自动选择存储方式：

- **内存存储**：小型的临时表通常存储在内存中（使用 MEMORY 存储引擎），访问速度非常快
- **磁盘存储**：当临时表的数据量超过 `tmp_table_size` 或 `max_heap_table_size` 配置参数时，MySQL 会自动将临时表转换为磁盘存储

可以通过以下命令查看临时表使用的存储引擎：


```sql
-- 查看临时表的创建语句，了解使用的存储引擎
SHOW CREATE TABLE temp_table_name;

-- 查看临时表的详细信息
SHOW TABLE STATUS LIKE 'temp_table_name';
```

### 9.5.连接关闭时的行为

临时表在连接关闭时会自动删除，这个行为在不同场景下的表现：

- **命令行客户端**：当你在 MySQL 命令行中输入 `exit` 或 `quit` 退出时，所有临时表会被自动删除
- **脚本执行**：当 PHP、Python 等脚本执行完成并关闭数据库连接时，临时表会被删除
- **应用程序**：当应用程序关闭数据库连接时，临时表会被删除
- **连接超时**：如果连接因为超时或其他原因断开，临时表也会被删除

**重要提示**：由于临时表会在连接关闭时自动删除，如果需要保留数据，应该使用普通表而不是临时表。
