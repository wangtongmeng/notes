## 1.引言

创建数据库是使用 MySQL 的第一步。在开始存储和管理数据之前，需要先创建一个数据库来存放这些数据。

## 2.数据库基础概念

### 2.1.什么是数据库？

数据库（Database）是存储和管理数据的容器。在 MySQL 中，一个数据库可以包含多个表（Table），每个表用于存储特定类型的数据。

**类比理解**：

- 数据库就像是一个文件夹
- 表就像是文件夹中的文件
- 数据就像是文件中的内容

### 2.2.数据库命名规范

在创建数据库时，建议遵循以下命名规范：

- 使用有意义的名称，便于识别和管理
- 只能包含字母、数字、下划线和连字符
- 不能使用 MySQL 保留关键字（如 SELECT、CREATE 等）
- 建议使用小写字母，避免大小写混用
- 名称长度建议不超过 64 个字符

## 3.使用 SQL 命令创建数据库

### 3.1.基本创建语法

在 MySQL 命令行中，使用 `CREATE DATABASE` 语句可以创建数据库。基本语法如下：


```sql
CREATE DATABASE 数据库名;
```

**命令说明**：

- `CREATE DATABASE`：创建数据库的 SQL 关键字
- `数据库名`：要创建的数据库名称，需要遵循命名规范

**示例：创建名为 `studentdb` 的数据库**

首先，需要登录到 MySQL 服务器：


```bash
mysql -u root -p
```

**执行过程**：

1. 输入命令后按回车
2. 系统会提示输入密码：`Enter password:`
3. 输入密码（输入时不会显示字符，这是正常的）
4. 按回车键确认

输入密码后，进入 MySQL 命令行环境，提示符会变成 `mysql>`。然后执行创建数据库的命令：


```sql
CREATE DATABASE studentdb;
```

**注意事项**：

- SQL 语句以分号（`;`）结尾，这是必需的
- 数据库名称不区分大小写，但建议统一使用小写
- 执行命令后按回车键

如果命令执行成功，会看到类似以下的提示：


```python
Query OK, 1 row affected (0.01 sec)
```

这表示数据库创建成功，用时 0.01 秒。

### 3.2.完整的创建语法

`CREATE DATABASE` 语句支持更多选项，完整语法如下：


```sql
CREATE DATABASE [IF NOT EXISTS] database_name
  [CHARACTER SET charset_name]
  [COLLATE collation_name];
```

**参数说明**：

- `IF NOT EXISTS`：如果数据库已存在，则不报错，直接跳过
- `CHARACTER SET`：指定数据库使用的字符集
- `COLLATE`：指定数据库使用的排序规则

### 3.3.指定字符集和排序规则

在创建数据库时，可以指定字符集和排序规则，这对于存储中文等多字节字符非常重要。

**示例：创建使用 UTF-8 字符集的数据库**


```sql
CREATE DATABASE myapp
  CHARACTER SET utf8mb4
  COLLATE utf8mb4_unicode_ci;
```

**参数说明**：

- `utf8mb4`：完整的 UTF-8 字符集，支持所有 Unicode 字符，包括 emoji 表情
- `utf8mb4_unicode_ci`：不区分大小写的 Unicode 排序规则

**常用字符集和排序规则**：

- `utf8mb4` + `utf8mb4_unicode_ci`：推荐用于现代应用，支持所有 Unicode 字符
- `utf8mb4` + `utf8mb4_general_ci`：性能稍好，但排序准确性略低
- `utf8` + `utf8_general_ci`：旧版本 UTF-8，不支持 4 字节字符（如 emoji）

### 3.4.使用 IF NOT EXISTS 避免错误

如果尝试创建一个已存在的数据库，MySQL 会报错。为了避免这种情况，可以使用 `IF NOT EXISTS` 子句：


```sql
CREATE DATABASE IF NOT EXISTS studentdb;
```

**执行结果**：

- 如果数据库不存在：创建数据库
- 如果数据库已存在：不报错，直接跳过

这个选项在脚本中特别有用，可以确保脚本可以重复执行而不会因为数据库已存在而失败。

### 3.5.验证数据库是否创建成功

创建数据库后，可以使用以下命令验证：

**查看所有数据库**：


```sql
SHOW DATABASES;
```

**输出示例**：


```python
+--------------------+
| Database           |
+--------------------+
| information_schema |
| mysql              |
| performance_schema |
| studentdb          |
| sys                |
+--------------------+
```

可以看到 `studentdb` 已经出现在数据库列表中。

**查看数据库的详细信息**：


```sql
SHOW CREATE DATABASE studentdb;
```

这个命令会显示创建数据库时使用的完整 SQL 语句，包括字符集和排序规则等信息。

## 4.常见问题

### 4.1.创建数据库时提示权限不足

**问题**：执行 `CREATE DATABASE` 时提示 `Access denied`

**解决方案**：

1. 使用具有 `CREATE` 权限的用户，或使用 `root` 用户

2. 为当前用户授予创建数据库的权限：


   ```sql
   GRANT CREATE ON *.* TO 'username'@'localhost'; FLUSH PRIVILEGES;
   ```

### 4.2.数据库名称包含特殊字符

**问题**：数据库名称包含空格或特殊字符导致创建失败

**解决方案**：

- 使用反引号（`）包裹数据库名称：


  ```sql
  CREATE DATABASE `my database`;
  ```

- 但建议避免使用特殊字符，改用下划线：


  ```sql
  CREATE DATABASE my_database;
  ```

### 4.3.字符集不匹配导致乱码

**问题**：创建数据库后，存储的中文数据出现乱码

**解决方案**：

- 创建数据库时明确指定字符集为

   

  ```
  utf8mb4
  ```

  ：


  ```sql
  CREATE DATABASE mydb CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
  ```

- 确保客户端连接时也使用正确的字符集

### 4.4.数据库已存在错误

**问题**：重复执行创建命令时提示数据库已存在

**解决方案**：

- 使用

   

  ```
  IF NOT EXISTS
  ```

   

  子句：


  ```sql
  CREATE DATABASE IF NOT EXISTS mydb;
  ```

- 或者先检查数据库是否存在，再决定是否创建
