## 1.引言

连接到 MySQL 服务器后，你可能会发现系统中有多个数据库。在进行任何数据操作之前，需要先明确指定要使用哪个数据库。本文介绍如何在 MySQL 中选择和切换数据库，这是进行后续数据操作的基础步骤。

## 2.为什么需要选择数据库？

MySQL 服务器可以同时管理多个数据库，每个数据库都是独立的数据存储空间。当你执行 SQL 语句时，MySQL 需要知道这些操作应该作用在哪个数据库上。

**类比理解**：

- 就像在文件系统中，你需要先进入某个文件夹，才能在该文件夹中创建或操作文件
- 选择数据库就像是"进入"某个数据库，之后的所有操作都会在这个数据库中进行

## 3.使用 USE 语句选择数据库

### 3.1.基本语法

在 MySQL 命令行中，使用 `USE` 语句来选择要操作的数据库：


```sql
USE 数据库名;
```

**参数说明**：

- `数据库名`：要选择的数据库名称（区分大小写，建议使用小写）

**注意事项**：

- 语句末尾必须加分号 `;`
- 数据库名如果包含特殊字符或空格，需要用反引号 ``` 括起来
- 选择数据库后，后续的 SQL 操作都会在指定的数据库上执行

### 3.2.基本使用示例

假设你已经连接到 MySQL 服务器，现在要选择一个名为 `mydb` 的数据库：


```sql
USE mydb;
```

执行成功后，MySQL 会返回确认信息：


```python
Database changed
```

看到这个提示，说明你已经成功选择了 `mydb` 数据库。

### 3.3.查看当前使用的数据库

选择数据库后，可以使用以下命令查看当前正在使用的数据库：


```sql
SELECT DATABASE();
```

**示例输出**：


```python
+------------+
| DATABASE() |
+------------+
| mydb       |
+------------+
1 row in set (0.00 sec)
```

### 3.4.查看所有可用的数据库

在选择数据库之前，你可能想先查看服务器上有哪些数据库可用：


```sql
SHOW DATABASES;
```

**示例输出**：


```python
+--------------------+
| Database           |
+--------------------+
| information_schema |
| mydb               |
| testdb             |
| mysql              |
| performance_schema |
+--------------------+
5 rows in set (0.00 sec)
```

**说明**：

- `information_schema`、`mysql`、`performance_schema` 是 MySQL 系统数据库，通常不需要直接操作
- `mydb`、`testdb` 等是用户创建的数据库

## 4.完整的操作流程示例

下面演示一个完整的操作流程：连接 MySQL、查看数据库列表、选择数据库、验证选择结果。

### 4.1.连接 MySQL 服务器

首先，使用命令行连接到 MySQL 服务器：


```bash
mysql -u root -p
```

输入密码后，进入 MySQL 命令行环境，提示符变为 `mysql>`。

### 4.2.查看数据库列表


```sql
mysql> SHOW DATABASES;
+--------------------+
| Database           |
+--------------------+
| information_schema |
| mydb               |
| testdb             |
| mysql              |
+--------------------+
4 rows in set (0.00 sec)
```

### 4.3.选择目标数据库

假设我们要使用 `testdb` 数据库：


```sql
mysql> USE testdb;
Database changed
```

### 4.4.验证当前数据库

确认当前使用的数据库：


```sql
mysql> SELECT DATABASE();
+------------+
| DATABASE() |
+------------+
| testdb     |
+------------+
1 row in set (0.00 sec)
```

### 4.5.查看数据库中的表

选择数据库后，可以查看该数据库中有哪些表：


```sql
mysql> SHOW TABLES;
+------------------+
| Tables_in_testdb |
+------------------+
| users            |
| products         |
| orders           |
+------------------+
3 rows in set (0.00 sec)
```

## 5.在连接时直接指定数据库

除了先连接再选择数据库，还可以在连接 MySQL 时直接指定要使用的数据库。

### 5.1.使用 -D 参数

在连接命令中使用 `-D` 参数可以直接指定数据库：


```bash
mysql -u 用户名 -p -D 数据库名
```

**参数说明**：

- `-u`：指定用户名
- `-p`：提示输入密码
- `-D`：指定要使用的数据库（注意是大写的 D）

### 5.2.使用示例

假设要连接并直接使用 `mydb` 数据库：


```bash
mysql -u root -p -D mydb
```

输入密码后，你会直接进入 `mydb` 数据库，无需再执行 `USE` 语句。

**验证**：


```sql
mysql> SELECT DATABASE();
+------------+
| DATABASE() |
+------------+
| mydb       |
+------------+
1 row in set (0.00 sec)
```

### 5.3.适用场景

**使用 `-D` 参数的优势**：

- 适合明确知道要使用哪个数据库的场景
- 可以简化操作流程，一步到位
- 在编写脚本时特别有用

**使用 `USE` 语句的优势**：

- 更灵活，可以在连接后根据情况切换数据库
- 适合需要操作多个数据库的场景

## 6.常见错误和解决方法

### 6.1.数据库不存在

如果尝试选择一个不存在的数据库，会收到错误提示：


```sql
mysql> USE nonexistent_db;
ERROR 1049 (42000): Unknown database 'nonexistent_db'
```

**解决方法**：

1. 使用 `SHOW DATABASES;` 查看所有可用的数据库
2. 确认数据库名称拼写是否正确（注意大小写）
3. 如果数据库确实不存在，需要先创建数据库

### 6.2.权限不足

如果当前用户没有访问某个数据库的权限，会收到权限错误：


```sql
mysql> USE restricted_db;
ERROR 1044 (42000): Access denied for user 'username'@'localhost' to database 'restricted_db'
```

**解决方法**：

1. 使用具有足够权限的用户（如 root）连接
2. 或者请数据库管理员为当前用户授予相应权限

### 6.3.数据库名称包含特殊字符

如果数据库名称包含特殊字符或空格，需要使用反引号：


```sql
-- 错误示例
mysql> USE my-database;
ERROR 1064 (42000): You have an error in your SQL syntax

-- 正确示例
mysql> USE `my-database`;
Database changed
```

## 7.使用 Python 脚本选择数据库

在实际开发中，我们经常需要通过编程语言来操作数据库。下面介绍如何使用 Python 的 `pymysql` 库来选择数据库。

### 7.1.安装 pymysql

首先需要安装 `pymysql` 库：


```bash
pip install pymysql
```

### 7.2.连接时指定数据库

在建立连接时直接指定要使用的数据库：


```python
import pymysql

# 数据库连接配置
host = 'localhost'      # 数据库服务器地址
user = 'root'           # 用户名
password = 'your_password'  # 密码
database = 'mydb'       # 要使用的数据库名

# 建立连接并指定数据库
conn = pymysql.connect(
    host=host,
    user=user,
    password=password,
    database=database  # 直接指定数据库
)

# 创建游标对象
cursor = conn.cursor()

# 执行查询
cursor.execute("SELECT DATABASE()")
result = cursor.fetchone()
print(f"当前使用的数据库: {result[0]}")

# 关闭连接
cursor.close()
conn.close()
```

### 7.3.连接后切换数据库

也可以先建立连接，然后使用 SQL 语句切换数据库：


```python
import pymysql

# 建立连接（不指定数据库）
conn = pymysql.connect(
    host='localhost',
    user='root',
    password='your_password'
)

cursor = conn.cursor()

# 使用 USE 语句选择数据库
cursor.execute("USE mydb")

# 验证当前数据库
cursor.execute("SELECT DATABASE()")
result = cursor.fetchone()
print(f"当前使用的数据库: {result[0]}")

# 关闭连接
cursor.close()
conn.close()
```

### 7.4.错误处理示例

在实际应用中，应该添加错误处理：


```python
import pymysql

try:
    # 建立连接
    conn = pymysql.connect(
        host='localhost',
        user='root',
        password='your_password',
        database='mydb'
    )
    
    cursor = conn.cursor()
    print("数据库连接成功")
    
    # 执行操作
    cursor.execute("SHOW TABLES")
    tables = cursor.fetchall()
    print(f"数据库中的表: {tables}")
    
except pymysql.Error as e:
    print(f"数据库操作错误: {e}")
    
finally:
    if 'cursor' in locals():
        cursor.close()
    if 'conn' in locals():
        conn.close()
    print("连接已关闭")
```

## 8.实践建议

### 8.1.数据库命名规范

- 使用有意义的数据库名称，便于识别和管理
- 建议使用小写字母、数字和下划线
- 避免使用 MySQL 保留字作为数据库名
- 如果必须使用特殊字符，记得用反引号括起来

### 8.2.操作前确认数据库

在执行重要操作（如删除表、修改数据）前，建议先确认当前使用的数据库：


```sql
SELECT DATABASE();
```

这样可以避免误操作其他数据库。

### 8.3.使用脚本时的最佳实践

在编写脚本时：

- 明确指定要使用的数据库，避免依赖默认设置
- 添加错误处理，检查数据库是否存在
- 在操作前验证数据库连接和权限

### 8.4.多数据库操作

如果需要同时操作多个数据库：



- 可以在不同的连接中分别选择不同的数据库
- 或者在 SQL 语句中使用 `数据库名.表名` 的格式来跨数据库操作

**示例**：


```sql
-- 即使当前在 mydb 数据库中，也可以查询 testdb 数据库的表
SELECT * FROM testdb.users;
```

## 9.总结

本文介绍了在 MySQL 中选择数据库的方法：

1. **USE 语句**：在 MySQL 命令行中使用 `USE 数据库名;` 选择数据库
2. **连接时指定**：使用 `mysql -D 数据库名` 在连接时直接指定数据库
3. **查看数据库**：使用 `SHOW DATABASES;` 查看所有可用数据库
4. **验证选择**：使用 `SELECT DATABASE();` 查看当前使用的数据库
5. **编程语言操作**：在 Python 等编程语言中，可以在连接时指定数据库或使用 SQL 语句切换

**关键要点**：

- 选择数据库是进行数据操作的前提步骤
- 使用 `USE` 语句是最常用的方法
- 在连接时使用 `-D` 参数可以一步到位
- 操作前确认当前数据库，避免误操作
- 注意数据库名称的大小写和特殊字符处理

掌握数据库选择的方法后，你就可以开始进行表的创建、数据的查询和修改等操作了。在后续的学习中，我们将深入了解表的操作、数据的增删改查等更多内容。
