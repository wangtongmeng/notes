## 1.引言

在数据表中存储数据后，查询数据是最常用的操作。通过查询，我们可以从数据库中获取需要的信息，这是数据库的核心功能之一。

本文将从查询数据的基础概念开始，逐步介绍如何使用 `SELECT` 语句从 MySQL 数据表中查询数据，包括基本语法、条件筛选、排序、限制结果数量，以及如何通过命令行执行查询操作。

## 2.查询数据基础概念

### 2.1.什么是查询数据？

查询数据是指从数据表中检索和获取符合条件的数据记录。查询操作不会修改数据，只是读取和展示数据。

**类比理解**：

- 数据表就像是一个 Excel 表格
- 查询数据就像是在表格中查找和筛选特定的行
- 查询结果可以包含所有列，也可以只选择需要的列

### 2.2.查询数据的前提条件

在执行查询操作之前，需要满足以下条件：

- **数据表已存在**：必须先创建好数据表，才能查询其中的数据
- **表中有数据**：如果表中没有数据，查询结果将为空
- **已选择数据库**：需要先使用 `USE` 语句选择要查询的数据库

### 2.3.查询数据的方式

MySQL 提供了灵活的查询方式：

- **查询所有列**：使用 `*` 通配符获取所有列的数据
- **查询指定列**：只选择需要的列，提高查询效率
- **条件查询**：使用 `WHERE` 子句筛选符合条件的数据
- **排序查询**：使用 `ORDER BY` 子句对结果进行排序
- **限制数量**：使用 `LIMIT` 子句限制返回的记录数

## 3.SELECT 语句基本语法

### 3.1.基本语法格式

从 MySQL 数据表查询数据使用 `SELECT` 语句，基本语法如下：


```sql
SELECT column1, column2, ...
FROM table_name
[WHERE condition]
[ORDER BY column_name [ASC | DESC]]
[LIMIT number];
```

**参数说明**：

- `column1, column2, ...`：要查询的列名，使用 `*` 表示选择所有列

- `table_name`：要查询的数据表名称

- `WHERE condition`：可选的筛选条件，只返回符合条件的行

- ```
  ORDER BY column_name
  ```

  ：可选的排序子句，按指定列排序

  - `ASC`：升序排列（从小到大），默认值
  - `DESC`：降序排列（从大到小）

- `LIMIT number`：可选的数量限制，限制返回的行数

**语法说明**：

- 方括号 `[]` 表示可选部分，可以根据需要添加
- 多个子句可以组合使用，实现复杂的查询需求

### 3.2.最简单的查询

**查询所有列的所有数据**：


```sql
SELECT * FROM products;
```

这个查询会返回 `products` 表中的所有行和所有列。

**查询指定列的数据**：


```sql
SELECT product_name, price FROM products;
```

这个查询只返回 `product_name` 和 `price` 两列的数据。

## 4.基本查询示例

假设我们有一个 `products` 表，包含以下字段：

- `product_id`：产品ID（主键）
- `product_name`：产品名称
- `price`：价格
- `stock`：库存数量
- `category`：产品类别
- `created_at`：创建时间

### 4.1.查询所有数据

**示例：查询所有产品信息**


```sql
SELECT * FROM products;
```

这个查询会返回表中所有产品的完整信息。

### 4.2.查询指定列

**示例：只查询产品名称和价格**


```sql
SELECT product_name, price FROM products;
```

**输出示例**：


```python
+----------------+--------+
| product_name   | price  |
+----------------+--------+
| 笔记本电脑     | 5999.00|
| 智能手机       | 3999.00|
| 无线耳机       |  299.00|
+----------------+--------+
```

### 4.3.使用 WHERE 子句筛选数据

`WHERE` 子句用于指定筛选条件，只返回符合条件的行。

**示例：查询价格大于 1000 的产品**


```sql
SELECT * FROM products WHERE price > 1000;
```

**示例：查询特定类别的产品**


```sql
SELECT product_name, price FROM products WHERE category = '电子产品';
```

**示例：查询库存大于 0 的产品**


```sql
SELECT * FROM products WHERE stock > 0;
```

### 4.4.使用 ORDER BY 子句排序

`ORDER BY` 子句用于对查询结果进行排序。

**示例：按价格升序排列**


```sql
SELECT * FROM products ORDER BY price;
```

或者明确指定升序：


```sql
SELECT * FROM products ORDER BY price ASC;
```

**示例：按价格降序排列**


```sql
SELECT * FROM products ORDER BY price DESC;
```

**示例：按多个字段排序**


```sql
SELECT * FROM products ORDER BY category, price DESC;
```

这个查询会先按 `category` 排序，然后在同一类别内按 `price` 降序排列。

### 4.5.使用 LIMIT 子句限制数量

`LIMIT` 子句用于限制返回的记录数量。

**示例：只返回前 5 条记录**


```sql
SELECT * FROM products LIMIT 5;
```

**示例：跳过前 10 条，返回接下来的 5 条**


```sql
SELECT * FROM products LIMIT 10, 5;
```

这个语法表示：跳过前 10 条记录，然后返回接下来的 5 条记录。

## 5.条件查询进阶

### 5.1.比较运算符

在 `WHERE` 子句中可以使用各种比较运算符：

| 运算符       | 说明     | 示例           |
| ------------ | -------- | -------------- |
| `=`          | 等于     | `price = 100`  |
| `!=` 或 `<>` | 不等于   | `price != 100` |
| `>`          | 大于     | `price > 100`  |
| `<`          | 小于     | `price < 100`  |
| `>=`         | 大于等于 | `price >= 100` |
| `<=`         | 小于等于 | `price <= 100` |

**示例：查询价格在 100 到 1000 之间的产品**


```sql
SELECT * FROM products WHERE price >= 100 AND price <= 1000;
```

或者使用 `BETWEEN` 运算符：


```sql
SELECT * FROM products WHERE price BETWEEN 100 AND 1000;
```

### 5.2.逻辑运算符

可以使用逻辑运算符组合多个条件：

| 运算符 | 说明                 | 示例                        |
| ------ | -------------------- | --------------------------- |
| `AND`  | 并且，所有条件都满足 | `price > 100 AND stock > 0` |
| `OR`   | 或者，满足任一条件   | `price < 100 OR stock = 0`  |
| `NOT`  | 非，不满足条件       | `NOT price > 1000`          |

**示例：查询价格大于 500 且库存大于 0 的产品**


```sql
SELECT * FROM products WHERE price > 500 AND stock > 0;
```

**示例：查询价格小于 100 或库存为 0 的产品**


```sql
SELECT * FROM products WHERE price < 100 OR stock = 0;
```

### 5.3.LIKE 模糊查询

`LIKE` 运算符用于进行模糊匹配，通常配合通配符使用：

- `%`：匹配任意多个字符（包括零个字符）
- `_`：匹配单个字符

**示例：查询产品名称以"电脑"开头的产品**


```sql
SELECT * FROM products WHERE product_name LIKE '电脑%';
```

**示例：查询产品名称包含"手机"的产品**


```sql
SELECT * FROM products WHERE product_name LIKE '%手机%';
```

**示例：查询产品名称第二个字是"机"的产品**


```sql
SELECT * FROM products WHERE product_name LIKE '_机%';
```

### 5.4.IN 子句

`IN` 子句用于匹配多个值，相当于多个 `OR` 条件的简化写法。

**示例：查询特定 ID 的产品**


```sql
SELECT * FROM products WHERE product_id IN (1, 3, 5, 7);
```

**示例：查询特定类别的产品**


```sql
SELECT * FROM products WHERE category IN ('电子产品', '办公用品', '家居用品');
```

### 5.5.NULL 值查询

查询字段值为 `NULL` 的记录需要使用 `IS NULL` 或 `IS NOT NULL`。

**示例：查询类别为空的产品**


```sql
SELECT * FROM products WHERE category IS NULL;
```

**示例：查询类别不为空的产品**


```sql
SELECT * FROM products WHERE category IS NOT NULL;
```

**注意**：不能使用 `= NULL` 来查询空值，必须使用 `IS NULL`。

## 6.组合查询示例

在实际使用中，我们经常需要组合使用多个子句来实现复杂的查询需求。

### 6.1.WHERE + ORDER BY

**示例：查询价格大于 500 的产品，并按价格降序排列**


```sql
SELECT * FROM products 
WHERE price > 500 
ORDER BY price DESC;
```

### 6.2.WHERE + ORDER BY + LIMIT

**示例：查询价格最高的前 3 个产品**


```sql
SELECT * FROM products 
ORDER BY price DESC 
LIMIT 3;
```

### 6.3.复杂条件查询

**示例：查询特定类别中价格在 100 到 1000 之间，且库存大于 0 的产品**


```sql
SELECT product_name, price, stock 
FROM products 
WHERE category = '电子产品' 
  AND price BETWEEN 100 AND 1000 
  AND stock > 0
ORDER BY price ASC;
```

### 6.4.模糊查询 + 排序

**示例：查询名称包含"手机"的产品，按价格升序排列**


```sql
SELECT * FROM products 
WHERE product_name LIKE '%手机%' 
ORDER BY price ASC;
```

## 7.通过命令行查询数据

### 7.1.连接到 MySQL

首先需要连接到 MySQL 服务器：


```bash
mysql -u root -p
```

输入密码后，会看到 `mysql>` 提示符。

### 7.2.选择数据库

使用 `USE` 语句选择要查询的数据库：


```sql
USE my_database;
```

### 7.3.执行查询语句

在 `mysql>` 提示符下，直接输入 `SELECT` 语句并执行：


```sql
SELECT * FROM products;
```

**输出示例**：


```python

| product_id | product_name   | price  | stock | category   | created_at |

|          3 | 无线耳机       |  299.00|   100 | 电子产品   | 2024-01-17 |

3 rows in set (0.00 sec)

可以使用 `\G` 来改变输出格式，使结果更易读：

## 8.查询优化建议

### 8.1.只查询需要的列

避免使用 `SELECT *`，只查询实际需要的列，可以提高查询效率：


```sql
-- 不推荐
SELECT * FROM products;

-- 推荐
SELECT product_name, price FROM products;
```

### 8.2.使用索引字段进行筛选

如果经常需要按某个字段查询，应该在该字段上创建索引，可以大大提高查询速度。

### 8.3.合理使用 LIMIT

当查询结果可能很大时，使用 `LIMIT` 限制返回数量，避免一次性返回过多数据：


```sql
SELECT * FROM products ORDER BY created_at DESC LIMIT 10;
```

### 8.4.避免在 WHERE 子句中使用函数

在 `WHERE` 子句中对字段使用函数会导致无法使用索引，影响查询性能：


```sql
-- 不推荐
SELECT * FROM products WHERE YEAR(created_at) = 2024;

-- 推荐
SELECT * FROM products WHERE created_at >= '2024-01-01' AND created_at < '2025-01-01';
```

### 8.5.使用 EXPLAIN 分析查询

使用 `EXPLAIN` 命令可以查看 MySQL 如何执行查询，帮助优化查询性能：


```sql
EXPLAIN SELECT * FROM products WHERE price > 1000;
```

## 9.常见问题与注意事项

### 9.1.字符串比较大小写

MySQL 的字符串比较默认是不区分大小写的，但可以通过 `BINARY` 关键字实现区分大小写的比较：


```sql
-- 不区分大小写（默认）
SELECT * FROM products WHERE product_name = 'laptop';

-- 区分大小写
SELECT * FROM products WHERE BINARY product_name = 'Laptop';
```

### 9.2.日期格式

查询日期时，建议使用标准的日期格式 `'YYYY-MM-DD'`：


```sql
SELECT * FROM products WHERE created_at = '2024-01-15';
```

### 9.3.数值精度

使用 `DECIMAL` 类型存储价格等需要精确计算的数值，避免使用 `FLOAT` 或 `DOUBLE` 可能带来的精度问题。

### 9.4.空字符串与 NULL

空字符串 `''` 和 `NULL` 是不同的：

- 空字符串是长度为 0 的字符串，是一个实际的值
- `NULL` 表示没有值，是未知或未定义的状态

**示例**：


```sql
-- 查询类别为空字符串的记录
SELECT * FROM products WHERE category = '';

-- 查询类别为 NULL 的记录
SELECT * FROM products WHERE category IS NULL;
```
