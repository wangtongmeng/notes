# MySQL 运算符

## 1.引言

运算符是 SQL 语句中用于执行各种计算和比较操作的重要工具。在 MySQL 中，运算符可以帮助我们进行数学运算、数据比较、逻辑判断等操作。

本文介绍 MySQL 中常用的运算符类型，包括算术运算符、比较运算符、逻辑运算符和位运算符，以及它们的优先级规则。掌握这些运算符的使用方法，能够让我们编写更灵活、更强大的 SQL 查询语句。

## 2.算术运算符

算术运算符用于执行基本的数学运算，包括加法、减法、乘法、除法等。

### 2.1.支持的算术运算符

MySQL 支持以下算术运算符：

| 运算符       | 作用 | 说明               |
| ------------ | ---- | ------------------ |
| `+`          | 加法 | 两个数相加         |
| `-`          | 减法 | 两个数相减         |
| `*`          | 乘法 | 两个数相乘         |
| `/` 或 `DIV` | 除法 | 两个数相除         |
| `%` 或 `MOD` | 取余 | 返回除法运算的余数 |

**重要提示**：在进行除法运算和取余运算时，如果除数为 0，MySQL 会返回 `NULL` 而不是报错。

### 2.2.加法运算

使用 `+` 运算符进行加法运算：


```sql
SELECT 15 + 8;
```

执行结果：


```python
+--------+
| 15 + 8 |
+--------+
|     23 |
+--------+
```

### 2.3.减法运算

使用 `-` 运算符进行减法运算：


```sql
SELECT 20 - 7;
```

执行结果：


```python
+--------+
| 20 - 7 |
+--------+
|     13 |
+--------+
```

### 2.4.乘法运算

使用 `*` 运算符进行乘法运算：


```sql
SELECT 6 * 7;
```

执行结果：


```python
+-------+
| 6 * 7 |
+-------+
|    42 |
+-------+
```

### 2.5.除法运算

使用 `/` 运算符进行除法运算，结果会保留小数：


```sql
SELECT 15 / 4;
```

执行结果：


```python
+--------+
| 15 / 4 |
+--------+
| 3.7500 |
+--------+
```

### 2.6.整数除法

使用 `DIV` 运算符进行整数除法，只返回商的整数部分：


```sql
SELECT 15 DIV 4;
```

执行结果：


```python
+----------+
| 15 DIV 4 |
+----------+
|        3 |
+----------+
```

### 2.7.取余运算

使用 `%` 或 `MOD` 运算符进行取余运算：


```sql
SELECT 15 MOD 4;
```

执行结果：


```python
+----------+
| 15 MOD 4 |
+----------+
|        3 |
+----------+
```

也可以使用 `%` 符号：


```sql
SELECT 15 % 4;
```

## 3.比较运算符

比较运算符用于比较两个值的大小或是否相等，常用于 `WHERE` 子句和条件判断中。

### 3.1.支持的比较运算符

MySQL 支持以下比较运算符：

| 运算符              | 描述       | 说明                                     |
| ------------------- | ---------- | ---------------------------------------- |
| `=`                 | 等于       | 判断两个值是否相等                       |
| `<>` 或 `!=`        | 不等于     | 判断两个值是否不相等                     |
| `>`                 | 大于       | 判断左边的值是否大于右边的值             |
| `<`                 | 小于       | 判断左边的值是否小于右边的值             |
| `<=`                | 小于等于   | 判断左边的值是否小于或等于右边的值       |
| `>=`                | 大于等于   | 判断左边的值是否大于或等于右边的值       |
| `BETWEEN`           | 在范围内   | 判断值是否在指定的两个值之间（包含边界） |
| `NOT BETWEEN`       | 不在范围内 | 判断值是否不在指定的两个值之间           |
| `IN`                | 在集合中   | 判断值是否在指定的值列表中               |
| `NOT IN`            | 不在集合中 | 判断值是否不在指定的值列表中             |
| `<=>`               | 安全等于   | 可以安全地比较 NULL 值                   |
| `LIKE`              | 模糊匹配   | 使用通配符进行模式匹配                   |
| `REGEXP` 或 `RLIKE` | 正则匹配   | 使用正则表达式进行匹配                   |
| `IS NULL`           | 为空       | 判断值是否为 NULL                        |
| `IS NOT NULL`       | 不为空     | 判断值是否不为 NULL                      |

**比较结果说明**：

- 比较结果为真时，返回 `1`
- 比较结果为假时，返回 `0`
- 比较结果不确定时（如涉及 NULL），返回 `NULL`

### 3.2.等于运算符

使用 `=` 运算符判断两个值是否相等：


```sql
SELECT 5 = 5;
```

执行结果：


```python
+-------+
| 5 = 5 |
+-------+
|     1 |
+-------+
```

当比较涉及 NULL 值时，结果会返回 NULL：


```sql
SELECT NULL = NULL;
```

执行结果：


```python
+-------------+
| NULL = NULL |
+-------------+
|        NULL |
+-------------+
```

### 3.3.不等于运算符

使用 `<>` 或 `!=` 运算符判断两个值是否不相等：


```sql

```

执行结果：


```python
+--------+

+--------+
|      1 |
+--------+
```

### 3.4.安全等于运算符

```sql
SELECT NULL <=> NULL;
```

执行结果：


```python
+---------------+
| NULL <=> NULL |
+---------------+
|             1 |
+---------------+
```


```sql
SELECT 5 <=> NULL;
```

执行结果：


```python
+-----------+
| 5 <=> NULL |
+-----------+
|         0 |
+-----------+
```

## 4.逻辑运算符

逻辑运算符用于组合多个条件，进行逻辑判断。

### 4.1.支持的逻辑运算符

MySQL 支持以下逻辑运算符：

| 运算符        | 作用     | 说明                     |
| ------------- | -------- | ------------------------ |
| `NOT` 或 `!`  | 逻辑非   | 对表达式取反             |
| `AND` 或 `&&` | 逻辑与   | 两个条件都为真时返回真   |
| `OR` 或 `||`  | 逻辑或   | 至少一个条件为真时返回真 |
| `XOR`         | 逻辑异或 | 两个条件不同时返回真     |

**逻辑运算结果**：

- 表达式为真时，返回 `1`
- 表达式为假时，返回 `0`

### 4.2.AND 运算符

`AND` 运算符表示逻辑与，只有当两个条件都为真时，结果才为真：


```sql
SELECT 5 > 3 AND 8 > 6;
```

执行结果：


```python
+------------------+
| 5 > 3 AND 8 > 6 |
+------------------+
|                1 |
+------------------+
```


```sql
SELECT 5 > 3 AND 2 > 6;
```

执行结果：


```python
+------------------+
| 5 > 3 AND 2 > 6 |
+------------------+
|                0 |
+------------------+
```

### 4.3.OR 运算符

`OR` 运算符表示逻辑或，只要有一个条件为真，结果就为真：


```sql
SELECT 5 > 3 OR 2 > 6;
```

执行结果：


```python
+-----------------+
| 5 > 3 OR 2 > 6 |
+-----------------+
|               1 |
+-----------------+
```


```sql
SELECT 2 > 5 OR 1 > 3;
```

执行结果：


```python
+-----------------+
| 2 > 5 OR 1 > 3 |
+-----------------+
|               0 |
+-----------------+
```

也可以使用 `||` 符号：


```sql
SELECT 3 > 1 || 4 > 2;
```

执行结果：


```python
+-----------------+
| 3 > 1 || 4 > 2 |
+-----------------+
|               1 |
+-----------------+
```

### 4.4.NOT 运算符

`NOT` 运算符表示逻辑非，对表达式取反：


```sql
SELECT NOT 5 > 3;
```

执行结果：


```python
+------------+
| NOT 5 > 3 |
+------------+
|          0 |
+------------+
```

也可以使用 `!` 符号：


```sql
SELECT !(2 > 5);
```

执行结果：


```python
+----------+
| !(2 > 5) |
+----------+
|        1 |
+----------+
```

### 4.5.XOR 运算符

`XOR` 运算符表示逻辑异或，当两个条件不同时返回真，相同时返回假：


```sql
SELECT 1 XOR 0;
```

执行结果：


```python
+---------+
| 1 XOR 0 |
+---------+
|       1 |
+---------+
```


```sql
SELECT 1 XOR 1;
```

执行结果：


```python
+---------+
| 1 XOR 1 |
+---------+
|       0 |
+---------+
```


```sql
SELECT 0 XOR 0;
```

执行结果：


```python
+---------+
| 0 XOR 0 |
+---------+
|       0 |
+---------+
```

也可以使用 `^` 符号进行异或运算：


```sql
SELECT 1 ^ 0;
```

执行结果：


```python
+-------+
| 1 ^ 0 |
+-------+
|     1 |
+-------+
```

## 5.位运算符

位运算符用于对二进制数进行位级别的运算。MySQL 会先将操作数转换为二进制，进行位运算，然后再将结果转换回十进制。

### 5.1.支持的位运算符

MySQL 支持以下位运算符：

| 运算符 | 作用     | 说明                                         |
| ------ | -------- | -------------------------------------------- |
| `&`    | 按位与   | 两个操作数的对应位都为1时，结果位为1         |
| `|`    | 按位或   | 两个操作数的对应位至少有一个为1时，结果位为1 |
| `^`    | 按位异或 | 两个操作数的对应位不同时，结果位为1          |
| `~`    | 按位取反 | 将操作数的每一位取反                         |
| `<<`   | 左移     | 将操作数的二进制位向左移动指定的位数         |
| `>>`   | 右移     | 将操作数的二进制位向右移动指定的位数         |

### 5.2.按位与运算符

`&` 运算符对两个数的二进制表示进行按位与运算：


```sql
SELECT 5 & 3;
```

执行结果：


```python
+-------+
| 5 & 3 |
+-------+
|     1 |
+-------+
```

**运算过程**：

- 5 的二进制：`101`
- 3 的二进制：`011`
- 按位与：`001`（对应位都为1时结果为1）
- 结果：`1`

### 5.3.按位或运算符

`|` 运算符对两个数的二进制表示进行按位或运算：


```sql
SELECT 5 | 3;
```

执行结果：


```python
+-------+
| 5 | 3 |
+-------+
|     7 |
+-------+
```

**运算过程**：

- 5 的二进制：`101`
- 3 的二进制：`011`
- 按位或：`111`（对应位至少有一个为1时结果为1）
- 结果：`7`

### 5.4.按位异或运算符

`^` 运算符对两个数的二进制表示进行按位异或运算：


```sql
SELECT 5 ^ 3;
```

执行结果：


```python
+-------+
| 5 ^ 3 |
+-------+
|     6 |
+-------+
```

**运算过程**：

- 5 的二进制：`101`
- 3 的二进制：`011`
- 按位异或：`110`（对应位不同时结果为1）
- 结果：`6`

### 5.5.按位取反运算符

`~` 运算符对操作数的每一位进行取反运算：


```sql
SELECT ~3;
```

执行结果：


```python
+----------------------+
| ~3                   |
+----------------------+
| 18446744073709551612 |
+----------------------+
```

**说明**：由于 MySQL 使用 64 位整数，取反操作会得到一个很大的无符号整数。

### 5.6.左移运算符

`<<` 运算符将操作数的二进制位向左移动指定的位数，右边补0：


```sql
SELECT 4 << 2;
```

执行结果：


```python
+--------+
| 4 << 2 |
+--------+
|     16 |
+--------+
```

**运算过程**：

- 4 的二进制：`100`
- 左移2位：`10000`（相当于在右边补两个0）
- 结果：`16`（相当于 4 × 2² = 16）

### 5.7.右移运算符

`>>` 运算符将操作数的二进制位向右移动指定的位数：


```sql
SELECT 16 >> 2;
```

执行结果：


```python
+---------+
| 16 >> 2 |
+---------+
|       4 |
+---------+
```

**运算过程**：

- 16 的二进制：`10000`
- 右移2位：`100`（相当于去掉右边两位）
- 结果：`4`（相当于 16 ÷ 2² = 4）

## 6.运算符优先级

当 SQL 语句中包含多个运算符时，MySQL 会按照运算符的优先级顺序进行计算。了解运算符优先级有助于编写正确的 SQL 语句。

### 6.1.优先级规则

运算符的优先级从高到低大致如下：

1. **最高优先级**：`!`、`BINARY`、`COLLATE`
2. **一元运算符**：`~`（按位取反）、`-`（负号）、`+`（正号）
3. **乘除运算符**：`*`、`/`、`DIV`、`%`、`MOD`
4. **加减运算符**：`+`、`-`
5. **位移运算符**：`<<`、`>>`
6. **按位运算符**：`&`、`|`、`^`
7. **比较运算符**：`=`、`<=>`、`>=`、`>`、`<=`、`<`、`<>`、`!=`、`IS`、`LIKE`、`REGEXP`、`IN`
8. **逻辑运算符**：`NOT`、`!`、`AND`、`&&`、`OR`、`||`、`XOR`
9. **最低优先级**：`:=`（赋值运算符）

### 6.2.使用括号改变优先级

可以使用括号来改变运算符的执行顺序。括号内的表达式会优先计算，就像数学中的括号一样。

**示例：不使用括号**


```sql
SELECT 2 + 3 * 4;
```

执行结果：


```python
+-----------+
| 2 + 3 * 4 |
+-----------+
|        14 |
+-----------+
```

**计算过程**：

- 由于 `*` 的优先级高于 `+`，先计算 `3 * 4 = 12`
- 然后计算 `2 + 12 = 14`

**示例：使用括号改变顺序**


```sql
SELECT (2 + 3) * 4;
```

执行结果：


```python
+-------------+
| (2 + 3) * 4 |
+-------------+
|          20 |
+-------------+
```

**计算过程**：

- 括号内的 `2 + 3` 先计算，结果为 `5`
- 然后计算 `5 * 4 = 20`

### 6.3.优先级实际应用示例

在实际查询中，运算符优先级会影响查询结果。下面通过几个例子来说明：

**示例一：算术运算符优先级**


```sql
SELECT 10 + 5 * 2;
```

执行结果：


```python
+------------+
| 10 + 5 * 2 |
+------------+
|         20 |
+------------+
```

由于乘法优先级高于加法，先计算 `5 * 2 = 10`，再计算 `10 + 10 = 20`。

**示例二：逻辑运算符优先级**

在 `WHERE` 子句中，`AND` 的优先级高于 `OR`：


```sql
SELECT 1 = 1 OR 2 = 2 AND 3 = 4;
```

执行结果：


```python
+---------------------------+
| 1 = 1 OR 2 = 2 AND 3 = 4 |
+---------------------------+
|                         1 |
+---------------------------+
```

**计算过程**：

- 先计算 `2 = 2 AND 3 = 4`，结果为 `0`（因为 `3 = 4` 为假）
- 然后计算 `1 = 1 OR 0`，结果为 `1`

如果使用括号明确优先级：


```sql
SELECT (1 = 1 OR 2 = 2) AND 3 = 4;
```

执行结果：


```python
+-------------------------------+
| (1 = 1 OR 2 = 2) AND 3 = 4 |
+-------------------------------+
|                             0 |
+-------------------------------+
```

**计算过程**：

- 先计算括号内的 `1 = 1 OR 2 = 2`，结果为 `1`
- 然后计算 `1 AND 3 = 4`，结果为 `0`

**示例三：混合运算符优先级**


```sql
SELECT 5 + 3 * 2 > 10 AND 8 < 15;
```

执行结果：


```python
+---------------------------+
| 5 + 3 * 2 > 10 AND 8 < 15 |
+---------------------------+
|                         1 |
+---------------------------+
```

**计算过程**：

1. 先计算 `3 * 2 = 6`（乘法优先级最高）
2. 然后计算 `5 + 6 = 11`（加法）
3. 接着计算 `11 > 10`，结果为 `1`（比较运算符）
4. 计算 `8 < 15`，结果为 `1`（比较运算符）
5. 最后计算 `1 AND 1 = 1`（逻辑运算符）

### 6.4.优先级记忆技巧

记住运算符优先级可能有些困难，这里提供一些记忆技巧：

1. **数学运算顺序**：乘除优先于加减，这与数学中的运算顺序一致
2. **逻辑运算顺序**：`NOT` > `AND` > `OR`，可以理解为"非"最强，"与"次之，"或"最弱
3. **括号优先**：不确定优先级时，使用括号明确表达意图，这样代码更清晰易读

### 6.5.最佳实践建议

在实际编写 SQL 语句时，建议：

1. **使用括号明确意图**：即使运算符优先级已经正确，使用括号可以让代码更易读
2. **复杂表达式分段**：对于复杂的表达式，可以拆分成多个步骤或使用子查询
3. **测试验证**：不确定优先级时，先写简单的测试语句验证结果

**推荐写法**：


```sql
-- 使用括号，意图清晰
SELECT (price * quantity) + tax AS total
FROM orders
WHERE (status = 'active' OR status = 'pending') AND amount > 100;
```

**不推荐写法**：


```sql
-- 依赖优先级，可读性差
SELECT price * quantity + tax AS total
FROM orders
WHERE status = 'active' OR status = 'pending' AND amount > 100;
```
