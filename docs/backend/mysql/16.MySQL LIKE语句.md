## 1.引言

在数据库查询中，我们经常遇到这样的情况：需要查找包含特定字符或符合某种模式的记录，而不是完全精确的匹配。

**实际场景举例**：

- 查找所有邮箱地址中包含 "gmail" 的用户（如 `user@gmail.com`、`test@gmail.com`）
- 查找所有以 "张" 开头的姓名（如 "张三"、"张伟"、"张小明"）
- 查找所有产品名称中包含 "手机" 的商品（如 "智能手机"、"手机保护壳"、"手机充电器"）

如果使用等号 `=` 进行精确匹配，我们只能查找完全相同的记录。但在很多情况下，我们需要的是"模糊匹配"——只要包含某些字符或符合某种模式即可。

**LIKE 子句的作用**：
LIKE 子句是 MySQL 中用于在 WHERE 子句中进行模糊匹配的关键字。它允许我们根据模式来搜索字符串，而不是精确匹配。通过使用通配符，我们可以构建灵活的搜索模式，满足各种查询需求。

本文将从 LIKE 子句的基础概念开始，逐步介绍如何使用 LIKE 子句进行模糊查询，包括通配符的使用方法、各种匹配模式的实例，以及如何通过命令行来执行模糊查询操作。

## 2.LIKE 子句基础概念

### 2.1.什么是 LIKE 子句？

LIKE 子句是 SQL 中用于在 WHERE 子句中进行模糊匹配的关键字。它允许我们根据模式来搜索字符串，而不是精确匹配。

**类比理解**：

- 精确匹配（使用 `=`）就像用钥匙开锁，必须完全匹配才能打开
- 模糊匹配（使用 `LIKE`）就像用搜索框查找，输入部分关键词就能找到相关内容
- 例如：查找所有包含 "北京" 的地址，而不是只查找完全等于 "北京" 的记录

### 2.2.LIKE 子句与等号的区别

在 WHERE 子句中，我们可以使用等号 `=` 来设定精确的匹配条件，例如：


```sql
SELECT * FROM users WHERE email = 'admin@example.com';
```

这个查询只会返回邮箱完全等于 `admin@example.com` 的记录。

但是，如果我们想查找所有邮箱地址中包含 "example" 的记录，就需要使用 LIKE 子句：


```sql
SELECT * FROM users WHERE email LIKE '%example%';
```

这个查询会返回所有邮箱中包含 "example" 的记录，如 `admin@example.com`、`user@example.org` 等。

### 2.3.LIKE 子句的使用场景

LIKE 子句适用于以下场景：

- **部分匹配**：查找包含特定字符或字符串的记录
- **前缀匹配**：查找以特定字符开头的记录
- **后缀匹配**：查找以特定字符结尾的记录
- **模式匹配**：根据特定模式查找记录（如固定长度的字符串）

## 3.LIKE 子句基本语法

### 3.1.基本语法格式

在 SELECT 语句中使用 LIKE 子句的基本语法如下：


```sql
SELECT column1, column2, ...
FROM table_name
WHERE column_name LIKE pattern;
```

**参数说明**：

- `column1, column2, ...`：要查询的列名，使用 `*` 表示选择所有列
- `table_name`：要查询的数据表名称
- `column_name`：要应用 LIKE 子句的列名
- `pattern`：用于匹配的模式，可以包含通配符

**语法说明**：

- LIKE 子句必须放在 WHERE 子句中使用
- 模式字符串需要用单引号或双引号括起来
- 如果没有使用通配符，LIKE 子句与等号 `=` 的效果相同

### 3.2.LIKE 子句的扩展使用

LIKE 子句不仅可以用于 SELECT 语句，还可以用于其他 SQL 语句：

- **UPDATE 语句**：更新符合模式的数据
- **DELETE 语句**：删除符合模式的数据

**示例：在 UPDATE 中使用 LIKE**


```sql
UPDATE products 
SET status = 'discontinued' 
WHERE product_name LIKE '%旧款%';
```

**示例：在 DELETE 中使用 LIKE**


```sql
DELETE FROM logs 
WHERE log_message LIKE '%错误%';
```

## 4.通配符

LIKE 子句的强大之处在于可以使用通配符来构建灵活的匹配模式。MySQL 支持两种主要的通配符。

### 4.1.百分号通配符（%）

百分号 `%` 是 LIKE 子句中最常用的通配符，它表示零个、一个或多个字符。可以把它理解为"任意字符序列"。

**通配符说明**：

- `%` 可以出现在模式的任何位置（开头、中间、结尾）
- 可以单独使用，也可以与其他字符组合使用
- 类似于正则表达式中的 `*`（星号）或文件搜索中的 `*` 通配符
- `%` 可以匹配空字符串，也可以匹配很长的字符串

**类比理解**：

- 在文件搜索中，`*.txt` 表示所有以 `.txt` 结尾的文件
- 在 LIKE 中，`'%.txt'` 表示所有以 `.txt` 结尾的字符串
- `%` 就像是一个"万能字符"，可以代表任意内容

**常见模式示例**：

| 模式       | 匹配说明            | 示例匹配                       |
| ---------- | ------------------- | ------------------------------ |
| `'张%'`    | 以"张"开头的字符串  | 张三、张伟、张小明、张         |
| `'%com'`   | 以"com"结尾的字符串 | example.com、test.com、com     |
| `'%test%'` | 包含"test"的字符串  | test、testing、mytest、test123 |
| `'%'`      | 匹配所有字符串      | 所有记录（包括空字符串）       |
| `'%手机%'` | 包含"手机"的字符串  | 智能手机、手机保护壳、我的手机 |

### 4.2.下划线通配符（_）

下划线 `_` 表示匹配单个字符。它只能匹配一个字符，不能匹配零个或多个字符。

**通配符说明**：

- `_` 必须匹配一个字符，不能为空（这是与 `%` 的重要区别）
- 可以连续使用多个 `_` 来匹配固定长度的字符
- 类似于正则表达式中的 `.`（点号），但 `_` 在 LIKE 中更明确地表示"一个字符"
- 可以匹配任何单个字符：字母、数字、符号等

**类比理解**：

- 在文件搜索中，`file_1.txt` 中的 `_` 是实际字符
- 在 LIKE 中，`'file_1.txt'` 中的 `_` 是通配符，可以匹配 `fileA1.txt`、`fileB1.txt` 等
- `_` 就像是一个"占位符"，表示"这里必须有一个字符，但可以是任意字符"

**常见模式示例**：

| 模式       | 匹配说明                               | 示例匹配               | 不匹配                                      |
| ---------- | -------------------------------------- | ---------------------- | ------------------------------------------- |
| `'_a%'`    | 第二个字符为"a"的字符串                | cat、bat、hat、aabc    | abc（第一个字符不是任意字符）               |
| `'__test'` | 以任意两个字符开头，后跟"test"的字符串 | 12test、abtest、xytest | test（缺少两个字符）、atest（只有一个字符） |
| `'a_b'`    | 三个字符，中间是任意字符               | a1b、axb、a_b、a b     | ab（只有两个字符）、a12b（中间有两个字符）  |
| `'___'`    | 恰好三个字符的字符串                   | abc、123、a1b          | ab（只有两个字符）、abcd（有四个字符）      |

### 4.3.通配符组合使用

可以将 `%` 和 `_` 组合使用，创建更复杂的匹配模式。

**示例：组合模式**


```sql
SELECT * FROM products WHERE product_code LIKE 'A%_1';
```

这个模式匹配：

- 以 "A" 开头
- 后跟零个或多个字符（由 `%` 匹配）
- 然后是任意一个字符（由 `_` 匹配）
- 最后是 "1"

匹配示例：`A001`、`A1231`、`Aabc1` 等。

**示例：更复杂的组合模式**

查找用户名倒数第二个字符为 "n"，最后一个字符为任意字符的用户：


```sql
SELECT * FROM users WHERE username LIKE '%n_';
```

这个模式会匹配：

- `anb`（倒数第二个字符是 'n'，最后一个字符是 'b'）
- `xny`（倒数第二个字符是 'n'，最后一个字符是 'y'）
- `123n4`（倒数第二个字符是 'n'，最后一个字符是 '4'）

**注意**：这个模式 `'%n_'` 表示前面可以有任意多个字符，然后是 'n'，最后必须是一个任意字符。因此匹配的字符串必须以 'n' 后面跟一个字符结尾。

**示例：固定长度和模式组合**

查找商品编号为 6 位，格式为：字母 + 数字 + 字母 + 数字 + 字母 + 数字：

**注意**：MySQL 的 LIKE 不支持正则表达式的字符类（如 `[0-9]`、`[A-Z]`），需要使用 `REGEXP` 或 `RLIKE` 来实现复杂的模式匹配。


```sql
SELECT * FROM products 
WHERE product_code REGEXP '^[A-Za-z][0-9][A-Za-z][0-9][A-Za-z][0-9]$';
```

或者使用更灵活的方式，结合 LIKE 和 REGEXP：


```sql
SELECT * FROM products 
WHERE product_code LIKE '______' 
AND product_code REGEXP '^[A-Za-z][0-9][A-Za-z][0-9][A-Za-z][0-9]$';
```

这个查询会匹配如 `A1b2C3`、`X9y8Z7` 等格式的商品编号。

## 5.LIKE 子句实例

### 5.1.前缀匹配

查找以特定字符或字符串开头的记录。

**示例：查找以特定字母开头的姓名**

假设有一个 `employees` 表，包含员工信息。查找所有姓氏以 "李" 开头的员工：


```sql
SELECT * FROM employees WHERE last_name LIKE '李%';
```

**执行结果示例**：


```python
+----+-----------+----------+------------+
| id | last_name | first_name | department |
+----+-----------+----------+------------+
| 1  | 李        | 明        | 技术部     |
| 5  | 李        | 华        | 销售部     |
| 8  | 李        | 强        | 人事部     |
+----+-----------+----------+------------+
```

### 5.2.后缀匹配

查找以特定字符或字符串结尾的记录。

**示例：查找特定域名结尾的邮箱**

查找所有邮箱地址以 ".edu" 结尾的用户：


```sql
SELECT * FROM users WHERE email LIKE '%.edu';
```

这个查询会返回所有教育机构邮箱，如 `student@university.edu`、`teacher@college.edu` 等。

### 5.3.包含匹配

查找包含特定字符或字符串的记录。

**示例：查找包含特定关键词的产品**

假设有一个 `products` 表，查找所有产品名称中包含 "手机" 的产品：


```sql
SELECT * FROM products WHERE product_name LIKE '%手机%';
```

**执行结果示例**：


```python
+----+------------------+--------+--------+
| id | product_name     | price  | stock  |
+----+------------------+--------+--------+
| 3  | 智能手机         | 2999   | 50     |
| 7  | 手机保护壳       | 29     | 200    |
| 12 | 手机充电器       | 59     | 150    |
+----+------------------+--------+--------+
```

### 5.4.固定位置匹配

使用下划线通配符匹配固定位置的字符。

**示例：查找第二个字符为特定值的记录**

查找产品编号的第二个字符为 "2" 的所有产品：


```sql
SELECT * FROM products WHERE product_code LIKE '_2%';
```

这个查询会匹配如 `A2001`、`B2015`、`X2abc` 等产品编号。

**示例：查找固定长度的模式**

查找用户名为 5 个字符，且第三个字符为 "a" 的用户：


```sql
SELECT * FROM users WHERE username LIKE '__a__';
```

这个模式会匹配如 `baaab`、`12a34` 等用户名（第三个字符必须是 'a'）。

**注意**：如果查找第三个字符为 "m" 的用户名（如 `admin`），应使用 `'__m__'`；如果查找第三个字符为 "s" 的用户名（如 `basic`），应使用 `'__s__'`。

### 5.5.不区分大小写匹配

默认情况下，LIKE 子句的匹配行为受数据库字符集和排序规则（collation）的影响。在大多数情况下，MySQL 的 LIKE 匹配是不区分大小写的。

**示例：不区分大小写的匹配**


```sql
SELECT * FROM employees WHERE email LIKE '%company%';
```

这个查询会匹配包含 "company"、"Company"、"COMPANY"、"CoMpAnY" 等所有大小写组合的邮箱地址。

**明确指定排序规则**：

如果需要明确指定不区分大小写的匹配，可以使用 `COLLATE` 子句：


```sql
SELECT * FROM employees 
WHERE email LIKE '%company%' 
COLLATE utf8mb4_general_ci;
```

这里的 `utf8mb4_general_ci` 中：

- `ci` 表示 case-insensitive（不区分大小写）
- `cs` 表示 case-sensitive（区分大小写）

**区分大小写匹配**：

如果需要区分大小写，可以使用 `BINARY` 关键字或指定区分大小写的排序规则：


```sql
-- 方法一：使用 BINARY
SELECT * FROM employees WHERE email LIKE BINARY '%Company%';

-- 方法二：使用区分大小写的排序规则
SELECT * FROM employees 
WHERE email LIKE '%Company%' 
COLLATE utf8mb4_bin;
```

### 5.6.转义特殊字符

如果需要在模式中匹配通配符本身（`%` 或 `_`），需要使用转义字符。

**示例：查找包含百分号的记录**


```sql
SELECT * FROM discounts WHERE description LIKE '%\%%';
```

这里第一个 `%` 是通配符，`\%` 是转义后的百分号字符，最后一个 `%` 又是通配符。

**使用 ESCAPE 关键字指定转义字符**：


```sql
SELECT * FROM discounts WHERE description LIKE '%#%' ESCAPE '#';
```

这个查询使用 `#` 作为转义字符，查找包含百分号的记录。

## 8.性能优化建议

### 8.1.LIKE 查询的性能考虑

LIKE 查询，特别是以 `%` 开头的模式（如 `'%keyword'`），通常无法使用索引，可能导致全表扫描，影响查询性能。

**性能优化建议**：

1. **避免前导通配符**：如果可能，尽量使用 `'keyword%'` 而不是 `'%keyword'`，这样可以使用索引
2. **使用全文索引**：对于大量文本搜索，考虑使用 MySQL 的全文索引（FULLTEXT）
3. **限制结果集**：使用 `LIMIT` 子句限制返回的记录数

**示例：优化后的查询**


```sql
-- 可以使用索引（前缀匹配）
SELECT * FROM products WHERE product_name LIKE '手机%';

-- 无法使用索引（包含匹配）
SELECT * FROM products WHERE product_name LIKE '%手机%';

-- 使用 LIMIT 限制结果
SELECT * FROM products WHERE product_name LIKE '%手机%' LIMIT 20;
```

### 8.2.创建索引

对于经常用于 LIKE 查询的列，可以考虑创建索引：


```sql
-- 创建普通索引
CREATE INDEX idx_product_name ON products(product_name);

-- 创建前缀索引（只索引前几个字符）
CREATE INDEX idx_product_name_prefix ON products(product_name(10));
```

## 9.常见问题与注意事项

### 9.1.大小写敏感性

MySQL 的 LIKE 匹配默认情况下是不区分大小写的（取决于字符集和排序规则）。如果需要区分大小写，可以使用 `BINARY` 关键字：


```sql
-- 不区分大小写（默认）
SELECT * FROM users WHERE username LIKE 'admin%';

-- 区分大小写
SELECT * FROM users WHERE username LIKE BINARY 'Admin%';
```

### 9.2.空值处理

LIKE 子句无法匹配 `NULL` 值。如果需要查找 `NULL` 值，需要使用 `IS NULL`：


```sql
-- LIKE 无法匹配 NULL
SELECT * FROM users WHERE email LIKE '%@%';  -- 不会返回 email 为 NULL 的记录

-- 查找 NULL 值
SELECT * FROM users WHERE email IS NULL;
```

### 9.3.转义字符的使用

当需要匹配通配符本身时，记得使用转义字符：


```sql
-- 查找包含下划线的记录
SELECT * FROM files WHERE filename LIKE '%\_%';

-- 使用自定义转义字符
SELECT * FROM files WHERE filename LIKE '%#_%' ESCAPE '#';
```

## 10.实践建议

### 10.1.合理使用通配符

- **前缀匹配优先**：尽量使用 `'keyword%'` 而不是 `'%keyword%'`，以提高查询性能
- **避免过度使用**：不要过度使用通配符，特别是在大数据量的表中
- **明确匹配需求**：根据实际需求选择合适的通配符和模式

### 10.2.安全性考虑

在使用 LIKE 子句时，特别是在 Web 应用中，需要注意：

- **防止 SQL 注入**：始终使用预处理语句或转义函数
- **验证用户输入**：对用户输入的搜索关键词进行验证和清理
- **限制搜索范围**：不要允许用户搜索所有字段，限制可搜索的列

### 10.3.测试不同模式

在实际使用前，建议测试不同的匹配模式，确保查询结果符合预期：


```sql
-- 测试不同的模式
SELECT * FROM products WHERE name LIKE '手机%';      -- 前缀匹配
SELECT * FROM products WHERE name LIKE '%手机%';     -- 包含匹配
SELECT * FROM products WHERE name LIKE '%手机';      -- 后缀匹配
SELECT * FROM products WHERE name LIKE '_手机%';    -- 固定位置匹配
```
