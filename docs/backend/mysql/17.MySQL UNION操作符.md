## 1.引言

在数据库查询中，我们经常遇到这样的情况：需要将多个查询的结果合并在一起，形成一个统一的结果集。

**实际场景举例**：

- 将员工表和客户表中的所有姓名合并显示
- 将不同月份销售数据的结果汇总在一起
- 将多个数据表中相同类型的字段组合成一个列表

如果分别执行多个 `SELECT` 语句，我们只能得到多个独立的结果集。但在很多情况下，我们需要将这些结果合并成一个统一的结果集，方便查看和分析。

**UNION 操作符的作用**：
UNION 操作符是 MySQL 中用于合并多个 SELECT 语句结果的关键字。它可以将两个或多个查询的结果组合到一个结果集中，并自动去除重复的行。通过 UNION 操作符，我们可以灵活地组合来自不同表或不同条件的数据。

本文将从 UNION 操作符的基础概念开始，逐步介绍如何使用 UNION 操作符合并查询结果，包括基本语法、与 UNION ALL 的区别、各种使用场景的实例，以及如何通过命令行执行 UNION 查询操作。

## 2.UNION 操作符基础概念

### 2.1.什么是 UNION 操作符？

UNION 操作符是 SQL 中用于合并多个 SELECT 语句结果的关键字。它可以将两个或多个查询的结果组合到一个结果集中。

**类比理解**：

- 多个 SELECT 语句就像多个独立的篮子，每个篮子里装着不同的水果
- UNION 操作符就像把这些篮子里的水果倒进一个大篮子
- 如果两个篮子里有相同的水果（重复数据），UNION 会自动去除重复，只保留一个
- 例如：将员工表和客户表中的所有姓名合并成一个列表，相同的姓名只显示一次

### 2.2.UNION 操作符的特点

UNION 操作符具有以下特点：

- **合并结果**：将多个查询的结果合并成一个结果集
- **去除重复**：自动去除结果集中的重复行
- **列数相同**：每个 SELECT 语句必须返回相同数量的列
- **数据类型兼容**：对应位置的列必须具有兼容的数据类型
- **统一排序**：可以使用 ORDER BY 对合并后的结果进行排序

### 2.3.UNION 与多个独立查询的区别

假设我们有两个表：`employees`（员工表）和 `customers`（客户表），都包含姓名字段。

**使用多个独立查询**：


```sql
SELECT name FROM employees;
SELECT name FROM customers;
```

这样会得到两个独立的结果集，无法统一处理。

**使用 UNION 合并**：


```sql
SELECT name FROM employees
UNION
SELECT name FROM customers;
```

这样会得到一个合并后的结果集，包含所有不重复的姓名，方便统一查看和分析。

### 2.4.UNION 操作符的使用场景

UNION 操作符适用于以下场景：

- **合并相似数据**：将多个表中相同类型的字段合并显示
- **汇总统计**：将不同条件下的查询结果汇总在一起
- **数据整合**：将分散在不同表中的相关数据整合成一个结果集
- **去重合并**：需要合并数据并自动去除重复项的场景

## 3.UNION 操作符语法

### 3.1.基本语法格式

MySQL UNION 操作符的基本语法如下：


```sql
SELECT column1, column2, ...
FROM table1
WHERE condition1
UNION
SELECT column1, column2, ...
FROM table2
WHERE condition2
[ORDER BY column1, column2, ...];
```

**参数说明**：

- `column1, column2, ...`：要选择的列的名称，如果使用 `*` 表示选择所有列

- `table1, table2, ...`：要从中查询数据的表的名称

- `condition1, condition2, ...`：每个 SELECT 语句的过滤条件，是可选的

- ```
  ORDER BY
  ```

  ：可选的排序子句，用于指定合并后的结果集的排序顺序

  - 注意：ORDER BY 只能出现在最后一个 SELECT 语句之后，对整个合并结果进行排序

**语法说明**：

- 方括号 `[]` 表示可选部分，可以根据需要添加
- UNION 可以连接两个或多个 SELECT 语句
- 每个 SELECT 语句的列数和对应位置的数据类型必须相同或兼容

### 3.2.UNION 的规则要求

使用 UNION 操作符时，必须遵循以下规则：

1. **列数必须相同**：所有 SELECT 语句必须返回相同数量的列
2. **数据类型兼容**：对应位置的列必须具有兼容的数据类型（可以不同，但必须兼容）
3. **列名使用第一个查询**：合并后的结果集使用第一个 SELECT 语句的列名
4. **ORDER BY 位置**：ORDER BY 子句只能放在最后一个 SELECT 语句之后

**示例：列数不匹配的错误**：


```sql
-- 错误示例：第一个查询返回2列，第二个查询返回1列
SELECT id, name FROM employees
UNION
SELECT name FROM customers;  -- 这会报错
```

**正确示例**：


```sql
-- 正确：两个查询都返回1列
SELECT name FROM employees
UNION
SELECT name FROM customers;
```

- 结果按第一列（`first_name`）排序

## 5.UNION ALL 操作符

### 5.1.UNION 与 UNION ALL 的区别

UNION 和 UNION ALL 都可以合并多个查询结果，但它们有一个重要区别：

- **UNION**：合并结果并自动去除重复行
- **UNION ALL**：合并结果但保留所有行，包括重复行

**性能差异**：

- UNION 需要检查并去除重复行，性能相对较慢
- UNION ALL 直接合并结果，不检查重复，性能更快
- 如果确定没有重复数据或需要保留重复数据，使用 UNION ALL 更高效

### 5.2.UNION ALL 基本语法

UNION ALL 的语法与 UNION 完全相同，只是将 `UNION` 替换为 `UNION ALL`：


```sql
SELECT column1, column2, ...
FROM table1
WHERE condition1
UNION ALL
SELECT column1, column2, ...
FROM table2
WHERE condition2
[ORDER BY column1, column2, ...];
```

### 5.3.UNION ALL 使用示例

**示例：合并学生和教师的城市（保留重复）**


```sql
SELECT city FROM students
UNION ALL
SELECT city FROM teachers
ORDER BY city;
```

**执行结果**：


```python
city
----
北京
北京
北京
上海
上海
广州
```

**说明**：

- 使用 UNION ALL 保留了所有城市，包括重复的
- "北京" 出现了3次（学生表2次，教师表1次）
- "上海" 出现了2次（学生表1次，教师表1次）

**对比：使用 UNION（去重）**：


```sql
SELECT city FROM students
UNION
SELECT city FROM teachers
ORDER BY city;
```

**执行结果**：


```python
city
----
北京
上海
广州
```

**说明**：

- 使用 UNION 自动去除了重复的城市
- 每个城市只显示一次

### 5.4.何时使用 UNION ALL

在以下情况下，建议使用 UNION ALL：

- **确定没有重复数据**：如果两个查询的结果不会有重复，使用 UNION ALL 性能更好
- **需要保留重复数据**：如果需要显示所有数据，包括重复项，必须使用 UNION ALL
- **性能优先**：在数据量较大时，如果不需要去重，使用 UNION ALL 可以显著提高查询性能

## 8.UNION 操作符的注意事项

### 8.1.列数和数据类型要求

**重要规则**：

1. **列数必须相同**：所有 SELECT 语句必须返回相同数量的列
2. **数据类型兼容**：对应位置的列必须具有兼容的数据类型

**错误示例**：


```sql
-- 错误：第一个查询返回2列，第二个查询返回1列
SELECT id, name FROM websites
UNION
SELECT country FROM apps;  -- 这会报错
```

**正确示例**：


```sql
-- 正确：两个查询都返回1列
SELECT name FROM websites
UNION
SELECT app_name FROM apps;
```

### 8.2.列名使用规则

合并后的结果集使用第一个 SELECT 语句的列名：


```sql
SELECT name FROM websites
UNION
SELECT app_name FROM apps;
```

结果集中的列名是 `name`（来自第一个查询），而不是 `app_name`。

**使用别名统一列名**：


```sql
SELECT name AS item_name FROM websites
UNION
SELECT app_name AS item_name FROM apps;
```

这样结果集中的列名就是 `item_name`。

### 8.3.ORDER BY 的使用限制

ORDER BY 子句只能放在最后一个 SELECT 语句之后，对整个合并结果进行排序：

**正确示例**：


```sql
SELECT name FROM websites
UNION
SELECT app_name FROM apps
ORDER BY name;  -- 正确：放在最后
```

**错误示例**：


```sql
SELECT name FROM websites
ORDER BY name  -- 错误：不能在这里使用 ORDER BY
UNION
SELECT app_name FROM apps;
```

### 8.4.UNION 与 UNION ALL 的选择

**选择 UNION 的情况**：

- 需要去除重复数据
- 不确定是否有重复数据，但希望结果唯一
- 数据量不大，性能影响可接受

**选择 UNION ALL 的情况**：

- 确定没有重复数据，需要更好的性能
- 需要保留所有数据，包括重复项
- 数据量较大，性能要求高

**性能建议**：

- 如果不需要去重，优先使用 UNION ALL
- UNION ALL 的性能通常比 UNION 好，因为它不需要检查重复

## 11.常见问题解答

### 11.1.UNION 可以连接多少个 SELECT 语句？

UNION 可以连接两个或多个 SELECT 语句，理论上没有数量限制，但实际使用中建议不要连接过多，以免影响性能和可读性。

### 11.2.UNION 和 JOIN 有什么区别？

- **UNION**：垂直合并多个查询结果（行合并）
- **JOIN**：水平连接多个表的数据（列合并）

### 11.3.可以在 UNION 中使用聚合函数吗？

可以，但需要在每个 SELECT 语句中分别使用：


```sql
SELECT COUNT(*) AS total FROM websites
UNION
SELECT COUNT(*) AS total FROM apps;
```

### 11.4.UNION 会影响性能吗？

- UNION 需要去重，性能相对较慢
- UNION ALL 不去重，性能更快
- 数据量较大时，建议优先使用 UNION ALL（如果不需要去重）

### 11.5.如何处理 NULL 值？

UNION 会将 NULL 值视为相同的值，如果有多个 NULL，UNION 只会保留一个：


```sql
SELECT NULL AS value
UNION
SELECT NULL AS value;
```

结果只会返回一个 NULL。
