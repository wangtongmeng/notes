## 1.引言

数据表是数据库中存储数据的基本单位。在创建数据库后，下一步就是创建数据表来组织和存储具体的数据。

本文将从数据表的基础概念开始，逐步介绍如何使用 SQL 命令创建数据表，包括基本语法、字段定义、数据类型选择，以及如何通过命令行和编程语言来创建数据表。

## 2.数据表基础概念

### 2.1.什么是数据表？

数据表（Table）是数据库中存储数据的结构化容器，由行（Row）和列（Column）组成。

**类比理解**：

- 数据库就像是一个文件夹
- 数据表就像是文件夹中的 Excel 表格
- 行（记录）就像是表格中的每一行数据
- 列（字段）就像是表格中的每一列，定义了数据的类型和属性

### 2.2.创建数据表需要的信息

在创建数据表之前，需要明确以下信息：

- **表名**：数据表的名称，用于标识和访问这个表
- **字段名**：表中每一列的名称，用于标识不同的数据项
- **数据类型**：每个字段存储的数据类型，如整数、字符串、日期等

### 2.3.数据表命名规范

建议遵循以下命名规范：

- 使用有意义的名称，能够清楚表达表的用途
- 只能包含字母、数字、下划线和连字符
- 不能使用 MySQL 保留关键字（如 SELECT、CREATE、TABLE 等）
- 建议使用小写字母，多个单词用下划线分隔
- 名称长度建议不超过 64 个字符

## 3.创建数据表的基本语法

### 3.1.基本语法格式

创建数据表使用 `CREATE TABLE` 语句，基本语法如下：


```sql
CREATE TABLE table_name (
    column1 datatype,
    column2 datatype,
    ...
);
```

**参数说明**：

- `table_name`：要创建的数据表名称
- `column1`, `column2`：表中的列名（字段名）
- `datatype`：每个列的数据类型

### 3.2.简单示例

下面是一个创建商品信息表的示例：


```sql
CREATE TABLE products (
    product_id INT AUTO_INCREMENT PRIMARY KEY,
    product_name VARCHAR(100) NOT NULL,
    price DECIMAL(10, 2) NOT NULL,
    stock_quantity INT DEFAULT 0,
    created_at DATE
);
```

**字段说明**：

- `product_id`：商品编号，整数类型，自动递增，作为主键
- `product_name`：商品名称，变长字符串，不允许为空
- `price`：商品价格，精确小数类型，不允许为空
- `stock_quantity`：库存数量，整数类型，默认值为 0
- `created_at`：创建日期，日期类型

### 3.3.字段属性说明

在定义字段时，可以使用以下属性：

- **NOT NULL**：字段不允许为空，插入数据时必须提供值
- **DEFAULT**：设置字段的默认值，如果插入数据时未指定该字段，将使用默认值
- **AUTO_INCREMENT**：自动递增，通常用于主键字段，每次插入新记录时自动加 1
- **PRIMARY KEY**：定义主键，主键用于唯一标识表中的每一行记录

## 4.创建数据表的详细步骤

### 4.1.登录 MySQL

首先需要登录到 MySQL 服务器：


```bash
mysql -u root -p
```

输入密码后，进入 MySQL 命令行环境，提示符会变成 `mysql>`。

### 4.2.选择数据库

在创建数据表之前，需要先选择要使用的数据库：


```sql
USE database_name;
```

例如，选择名为 `shopdb` 的数据库：


```sql
USE shopdb;
```

执行成功后会显示：


```python
Database changed
```

### 4.3.创建数据表

选择数据库后，就可以创建数据表了。下面创建一个员工信息表的完整示例：


```sql
CREATE TABLE employees (
    emp_id INT UNSIGNED AUTO_INCREMENT,
    emp_name VARCHAR(50) NOT NULL,
    department VARCHAR(30) NOT NULL,
    salary DECIMAL(10, 2),
    hire_date DATE,
    is_active BOOLEAN DEFAULT TRUE,
    PRIMARY KEY (emp_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

**字段解析**：

- `emp_id`：员工编号，无符号整数，自动递增
- `emp_name`：员工姓名，变长字符串，不允许为空
- `department`：所属部门，变长字符串，不允许为空
- `salary`：薪资，精确小数类型
- `hire_date`：入职日期，日期类型
- `is_active`：是否在职，布尔类型，默认值为 TRUE
- `PRIMARY KEY (emp_id)`：将 `emp_id` 设置为主键

**表选项说明**：

- `ENGINE=InnoDB`：指定存储引擎为 InnoDB，支持事务和外键
- `DEFAULT CHARSET=utf8mb4`：设置默认字符集为 utf8mb4，支持完整的 Unicode 字符

### 4.4.使用 IF NOT EXISTS 避免错误

如果尝试创建一个已存在的数据表，MySQL 会报错。可以使用 `IF NOT EXISTS` 子句来避免这个问题：


```sql
CREATE TABLE IF NOT EXISTS employees (
    emp_id INT UNSIGNED AUTO_INCREMENT,
    emp_name VARCHAR(50) NOT NULL,
    PRIMARY KEY (emp_id)
);
```

**执行结果**：

- 如果表不存在：创建数据表
- 如果表已存在：不报错，直接跳过

这个选项在脚本中特别有用，可以确保脚本可以重复执行而不会因为表已存在而失败。

### 4.5.指定字符集和排序规则

在创建表时，可以指定字符集和排序规则，这对于存储中文等多字节字符非常重要：


```sql
CREATE TABLE customers (
    customer_id INT PRIMARY KEY,
    customer_name VARCHAR(50)
) CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

**参数说明**：

- `CHARACTER SET utf8mb4`：指定字符集为 utf8mb4，支持所有 Unicode 字符
- `COLLATE utf8mb4_unicode_ci`：指定排序规则，不区分大小写

**常用字符集和排序规则**：

- `utf8mb4` + `utf8mb4_unicode_ci`：推荐用于现代应用，支持所有 Unicode 字符
- `utf8mb4` + `utf8mb4_general_ci`：性能稍好，但排序准确性略低

### 4.6.验证数据表是否创建成功

创建数据表后，可以使用以下命令验证：


**输出示例**：


```python

+------------------+

+------------------+
```

**查看表结构**：

使用 `DESCRIBE` 或 `DESC` 命令可以查看表的结构：


```sql

```

或者使用：


```sql
DESC employees;
```

**输出示例**：


```python
+-----------+------------------+------+-----+---------+----------------+
| Field     | Type             | Null | Key | Default | Extra          |
+-----------+------------------+------+-----+---------+----------------+
| emp_id    | int(10) unsigned | NO   | PRI | NULL    | auto_increment |
| emp_name  | varchar(50)      | NO   |     | NULL    |                |
| department| varchar(30)      | NO   |     | NULL    |                |
| salary    | decimal(10,2)    | YES  |     | NULL    |                |
```

**字段信息说明**：

- **Field**：字段名称

- **Key**：键的类型（PRI 表示主键）
- **Default**：默认值
- **Extra**：额外信息（auto_increment 表示自动递增）

## 5.创建数据表的完整示例

### 5.1.创建学生信息表

下面创建一个学生信息表的完整示例，包含常用的字段类型和属性：


```sql
CREATE TABLE IF NOT EXISTS students (
    student_id INT UNSIGNED AUTO_INCREMENT,
    student_name VARCHAR(50) NOT NULL,
    gender CHAR(1) NOT NULL,
    age INT,
    email VARCHAR(100) NOT NULL,
    enrollment_date DATE,
    is_enrolled BOOLEAN DEFAULT TRUE,
    PRIMARY KEY (student_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

**字段解析**：

- `student_id`：学生编号，无符号整数，自动递增，作为主键
- `student_name`：学生姓名，变长字符串，不允许为空
- `gender`：性别，固定长度字符，不允许为空
- `age`：年龄，整数类型，允许为空
- `email`：电子邮箱，变长字符串，不允许为空
- `enrollment_date`：入学日期，日期类型，允许为空
- `is_enrolled`：是否在校，布尔类型，默认值为 TRUE
- `PRIMARY KEY (student_id)`：将 `student_id` 设置为主键

### 5.2.创建文章信息表

下面创建一个文章信息表的示例，展示更多字段类型的使用：


```sql
CREATE TABLE IF NOT EXISTS articles (
    article_id INT UNSIGNED AUTO_INCREMENT,
    article_title VARCHAR(200) NOT NULL,
    article_author VARCHAR(50) NOT NULL,
    publish_date DATE,
    content TEXT,
    view_count INT DEFAULT 0,
    is_published BOOLEAN DEFAULT FALSE,
    PRIMARY KEY (article_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

**字段解析**：

- `article_id`：文章编号，无符号整数，自动递增，作为主键
- `article_title`：文章标题，变长字符串，不允许为空
- `article_author`：文章作者，变长字符串，不允许为空
- `publish_date`：发布日期，日期类型，允许为空
- `content`：文章内容，文本类型，可以存储大量文本
- `view_count`：浏览次数，整数类型，默认值为 0
- `is_published`：是否已发布，布尔类型，默认值为 FALSE

**表选项说明**：

- `ENGINE=InnoDB`：使用 InnoDB 存储引擎，支持事务处理
- `DEFAULT CHARSET=utf8mb4`：默认字符集为 utf8mb4
- `COLLATE=utf8mb4_unicode_ci`：排序规则为 utf8mb4_unicode_ci，支持完整的 Unicode 字符排序

### 5.3.使用反引号保护表名和字段名

如果表名或字段名是 MySQL 的保留关键字，或者包含特殊字符，可以使用反引号（`）来保护：


```sql
CREATE TABLE IF NOT EXISTS `order_info` (
    `order_id` INT UNSIGNED AUTO_INCREMENT,
    `order_date` DATE NOT NULL,
    `total_amount` DECIMAL(10, 2) NOT NULL,
    `status` VARCHAR(20) DEFAULT 'pending',
    PRIMARY KEY (`order_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

**注意事项**：

- 虽然使用反引号可以避免关键字冲突，但建议尽量避免使用 MySQL 保留关键字作为表名或字段名
- 反引号在 MySQL 中是可选的，但使用它们可以提高代码的可读性和安全性

## 6.通过命令行创建数据表

### 6.1.登录 MySQL

输入密码后，如果连接成功，会看到 MySQL 提示符：


```python

```

### 6.2.选择数据库

在创建数据表之前，需要先选择要使用的数据库：


```sql

```

执行成功后会显示：


```python

```

### 6.3.创建数据表

在 MySQL 命令行中，可以逐行输入创建表的 SQL 语句。MySQL 会自动识别多行输入，直到遇到分号（`;`）才执行：


```sql
mysql> CREATE TABLE books (
    -> book_id INT NOT NULL AUTO_INCREMENT,
    -> book_title VARCHAR(150) NOT NULL,
    -> book_author VARCHAR(60) NOT NULL,
    -> publication_date DATE,
    -> price DECIMAL(8, 2),
    -> PRIMARY KEY (book_id)
    -> ) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
Query OK, 0 rows affected (0.15 sec)
```

**说明**：

- `mysql>` 是 MySQL 的命令提示符
- `->` 是换行提示符，表示命令还未结束，等待继续输入
- 输入分号（`;`）后，MySQL 会执行整个命令
- `Query OK, 0 rows affected (0.15 sec)` 表示命令执行成功

**注意事项**：

- MySQL 命令必须以分号（`;`）结尾
- `->` 是 MySQL 自动显示的换行提示符，不需要手动输入
- 如果输入错误，可以输入 `\c` 取消当前命令，然后重新输入

## 7.实践建议

### 7.1.表设计原则

在设计数据表时，建议遵循以下原则：

- **选择合适的字段类型**：根据实际需求选择合适的数据类型，避免浪费存储空间
- **使用主键**：每个表都应该有一个主键，用于唯一标识每一行记录
- **合理使用索引**：对于经常查询的字段，可以考虑添加索引以提高查询性能
- **设置合适的默认值**：为字段设置合理的默认值，可以简化数据插入操作
- **使用 NOT NULL 约束**：对于必须的字段，使用 NOT NULL 约束确保数据完整性

### 7.2.命名规范

建议遵循以下命名规范：

- **表名**：使用复数形式或描述性名称，如 `students`、`order_items`
- **字段名**：使用小写字母和下划线，如 `student_name`、`created_at`
- **主键**：通常使用 `表名_id` 的格式，如 `student_id`、`order_id`
- **布尔字段**：使用 `is_` 或 `has_` 前缀，如 `is_active`、`has_permission`

### 7.3.字符集选择

- **推荐使用 utf8mb4**：支持完整的 Unicode 字符，包括 emoji 等特殊字符

- 排序规则选择

  ：

  - `utf8mb4_unicode_ci`：推荐用于大多数场景，排序准确性高
  - `utf8mb4_general_ci`：性能稍好，但排序准确性略低

### 7.4.存储引擎选择

- **InnoDB**：推荐用于大多数场景，支持事务、外键、行级锁定
- **MyISAM**：适用于只读或读多写少的场景，但不支持事务和外键

### 7.5.使用 IF NOT EXISTS

在脚本中创建表时，建议使用 `IF NOT EXISTS` 子句，这样可以避免因表已存在而导致的错误，使脚本可以安全地重复执行。
