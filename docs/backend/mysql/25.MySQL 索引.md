## 1.引言

索引是 MySQL 数据库中非常重要的性能优化工具。合理使用索引可以显著提高数据库查询的速度，让数据库操作更加高效。

本文将从索引的基本概念开始，逐步介绍普通索引和唯一索引的创建、修改和删除方法，以及如何查看索引信息。通过实际示例，你将学会如何为表创建合适的索引，提升数据库查询性能。

## 2.索引基础概念

### 2.1.什么是索引？

MySQL 索引是一种特殊的数据结构，它存储了表中某些列的值以及指向这些值所在数据行的指针。通过索引，MySQL 可以快速定位到满足查询条件的数据，而不需要逐行扫描整个表。

**索引的作用**：

- **提高查询速度**：索引让数据库能够快速找到需要的数据，特别是在大型表中
- **加速排序**：索引可以帮助 MySQL 快速对结果进行排序
- **优化连接**：在表连接操作中，索引可以显著提高性能
- **保证唯一性**：唯一索引可以确保数据的唯一性约束

### 2.2.索引的类比理解

为了更好地理解索引，我们可以用生活中的例子来类比：

**类比一：书籍目录**

- 想象一下，如果没有目录，要在字典中查找一个字，需要逐页翻找
- 有了目录（索引），我们可以直接定位到对应的页码，快速找到目标
- MySQL 索引就像书籍的目录，帮助数据库快速定位数据

**类比二：图书馆检索系统**

- 图书馆的检索系统可以按书名、作者、分类号等多种方式查找图书
- 每种查找方式都相当于一个索引
- MySQL 也可以为不同的列创建索引，支持多种查询方式

### 2.3.索引的类型

MySQL 中的索引主要分为两类：

**单列索引**：

- 只包含单个列的索引
- 一个表可以有多个单列索引
- 例如：为姓名列创建索引，为年龄列创建索引

**组合索引**：

- 包含多个列的索引
- 适用于多列组合查询的场景
- 例如：为（姓名，年龄）两列创建组合索引

### 2.4.索引的工作原理

索引本质上也是一张表，这张表保存了主键与索引字段的值，并指向实体表中对应的记录。

**查询过程对比**：

**不使用索引**：

1. MySQL 需要逐行扫描整个表
2. 对每一行数据都检查是否满足查询条件
3. 数据量越大，查询时间越长

**使用索引**：

1. MySQL 首先在索引中查找满足条件的数据
2. 通过索引中的指针直接定位到对应的数据行
3. 大大减少了需要检查的数据行数

### 2.5.索引的注意事项

虽然索引能够提高查询性能，但也有一些需要注意的地方：

**优点**：

- 显著提高查询速度
- 加速表之间的连接
- 帮助实现数据的唯一性约束

**缺点**：

- 索引需要占用额外的存储空间
- 对表进行插入、更新和删除操作时，索引也需要维护，可能会影响这些操作的性能
- 过多或不合理的索引可能会导致性能下降，因此需要谨慎选择和规划索引

**使用建议**：

- 在经常用于查询条件的列上创建索引
- 在经常用于排序的列上创建索引
- 避免在频繁更新的列上创建过多索引
- 不要为数据量很小的表创建索引

## 3.普通索引

普通索引是最常用的索引类型，它没有任何限制，允许在索引列中插入重复值和空值。普通索引主要用于提高查询性能。

### 3.1.创建索引

使用 `CREATE INDEX` 语句可以创建普通索引。

**基本语法**：


```sql
CREATE INDEX index_name
ON table_name (column1 [ASC|DESC], column2 [ASC|DESC], ...);
```

**参数说明**：

- `CREATE INDEX`：创建索引的关键字
- `index_name`：索引的名称，在表中必须唯一
- `table_name`：要在哪个表上创建索引
- `(column1, column2, ...)`：要创建索引的列名，可以是一个或多个列
- `ASC` 和 `DESC`（可选）：指定索引的排序顺序，默认为升序（ASC）

**示例：为单列创建索引**

假设我们有一个名为 `products` 的表，包含商品信息，我们经常需要根据商品名称进行查询：


```sql
CREATE INDEX idx_product_name ON products (product_name);
```

这个语句在 `products` 表的 `product_name` 列上创建了一个名为 `idx_product_name` 的普通索引，这将有助于提高通过商品名称进行搜索的查询性能。

**示例：为多列创建组合索引**

如果经常需要同时根据商品名称和价格进行查询，可以创建组合索引：


```sql
CREATE INDEX idx_name_price ON products (product_name, price);
```

**注意事项**：

- 如果表中的数据量较大，创建索引可能需要一些时间
- 创建索引后，查询性能将会显著提高
- 索引名称应该具有描述性，便于识别和管理

### 3.2.使用 ALTER TABLE 添加索引

除了使用 `CREATE INDEX`，我们还可以使用 `ALTER TABLE` 命令在已有的表上创建索引。

**基本语法**：


```sql
ALTER TABLE table_name
ADD INDEX index_name (column1 [ASC|DESC], column2 [ASC|DESC], ...);
```

**参数说明**：

- `ALTER TABLE`：修改表结构的关键字
- `table_name`：要修改的表名
- `ADD INDEX`：添加索引的子句
- `index_name`：索引名称
- `(column1, column2, ...)`：要创建索引的列

**示例：为已有表添加索引**

假设我们有一个名为 `orders` 的表，现在要为订单日期列创建索引：


```sql
ALTER TABLE orders
ADD INDEX idx_order_date (order_date);
```

这个语句在 `orders` 表的 `order_date` 列上创建了一个名为 `idx_order_date` 的普通索引。

**示例：添加组合索引**

为订单表的客户ID和订单状态创建组合索引：


```sql
ALTER TABLE orders
ADD INDEX idx_customer_status (customer_id, order_status);
```

### 3.3.创建表时指定索引

在创建表的时候，可以直接在 `CREATE TABLE` 语句中指定索引，这样可以在创建表的同时创建索引。

**基本语法**：


```sql
CREATE TABLE table_name (
  column1 data_type,
  column2 data_type,
  ...,
  INDEX index_name (column1 [ASC|DESC], column2 [ASC|DESC], ...)
);
```

**参数说明**：

- `CREATE TABLE`：创建表的关键字
- `table_name`：要创建的表名
- `(column1 data_type, ...)`：定义表的列和数据类型
- `INDEX`：创建普通索引的关键字
- `index_name`：索引名称
- `(column1, column2, ...)`：要创建索引的列

**示例：创建表时同时创建索引**

创建一个商品分类表，并在分类名称列上创建索引：


```sql
CREATE TABLE categories (
  id INT PRIMARY KEY,
  category_name VARCHAR(100),
  description TEXT,
  INDEX idx_category_name (category_name)
);
```

这个语句创建了 `categories` 表，并在 `category_name` 列上创建了一个名为 `idx_category_name` 的普通索引。

**示例：创建表时创建多个索引**


```sql
CREATE TABLE users (
  id INT PRIMARY KEY,
  username VARCHAR(50),
  email VARCHAR(100),
  created_at DATETIME,
  INDEX idx_username (username),
  INDEX idx_created_at (created_at)
);
```

### 3.4.删除索引

当索引不再需要时，可以使用 `DROP INDEX` 语句删除索引。

**基本语法**：


```sql
DROP INDEX index_name ON table_name;
```

**参数说明**：

- `DROP INDEX`：删除索引的关键字
- `index_name`：要删除的索引名称
- `ON table_name`：指定要删除索引的表

**使用 ALTER TABLE 删除索引**：

也可以使用 `ALTER TABLE` 语句删除索引：


```sql
ALTER TABLE table_name
DROP INDEX index_name;
```

**示例：删除索引**

假设我们要删除 `products` 表上的 `idx_product_name` 索引：


```sql
DROP INDEX idx_product_name ON products;
```

或者使用 `ALTER TABLE`：


```sql
ALTER TABLE products
DROP INDEX idx_product_name;
```

**注意事项**：

- 如果索引不存在，执行删除命令会产生错误
- 删除索引前最好确认该索引是否存在
- 删除索引不会影响表中的数据，只是移除了索引结构

## 4.唯一索引

唯一索引与普通索引类似，但有一个重要的区别：唯一索引确保索引中的值是唯一的，不允许有重复值（NULL 值除外，NULL 可以出现多次）。

唯一索引既保证了数据的唯一性，又能提高查询性能。

### 4.1.创建唯一索引

使用 `CREATE UNIQUE INDEX` 语句可以创建唯一索引。

**基本语法**：


```sql
CREATE UNIQUE INDEX index_name
ON table_name (column1 [ASC|DESC], column2 [ASC|DESC], ...);
```

**参数说明**：

- `CREATE UNIQUE INDEX`：创建唯一索引的关键字组合
- `index_name`：唯一索引的名称，在表中必须唯一
- `table_name`：要在哪个表上创建唯一索引
- `(column1, column2, ...)`：要创建索引的列
- `ASC` 和 `DESC`（可选）：指定索引的排序顺序

**示例：创建唯一索引**

假设我们有一个名为 `customers` 的表，包含客户信息，我们希望确保每个客户的手机号码都是唯一的：


```sql
CREATE UNIQUE INDEX idx_phone ON customers (phone_number);
```

这个语句在 `customers` 表的 `phone_number` 列上创建了一个名为 `idx_phone` 的唯一索引。创建后，如果尝试插入重复的手机号码，MySQL 会拒绝该操作并报错。

**示例：为多列创建唯一索引**

如果希望确保两列的组合值是唯一的，可以创建多列唯一索引：


```sql
CREATE UNIQUE INDEX idx_user_email ON users (user_id, email);
```

### 4.2.使用 ALTER TABLE 添加唯一索引

可以使用 `ALTER TABLE` 命令来创建唯一索引。

**基本语法**：


```sql
ALTER TABLE table_name
ADD CONSTRAINT constraint_name UNIQUE (column1, column2, ...);
```

**参数说明**：

- `ALTER TABLE`：修改表结构的关键字
- `table_name`：要修改的表名
- `ADD CONSTRAINT`：添加约束的关键字
- `constraint_name`：约束名称，在表中必须唯一
- `UNIQUE (column1, column2, ...)`：指定要创建唯一索引的列

**示例：使用 ALTER TABLE 创建唯一索引**

假设我们有一个名为 `books` 的表，现在要为 ISBN 号列创建唯一索引，确保每本书的 ISBN 都是唯一的：


```sql
ALTER TABLE books
ADD CONSTRAINT uk_isbn UNIQUE (isbn);
```

这个语句在 `books` 表的 `isbn` 列上创建了一个名为 `uk_isbn` 的唯一索引。

**重要提示**：

- 如果表中已经存在重复的值，创建唯一索引将会失败
- 在创建唯一索引之前，需要先确保表中的数据没有重复值
- 可以使用 `SELECT` 语句检查是否有重复值

**示例：检查重复值**

在创建唯一索引前，可以先检查是否有重复：


```sql
SELECT isbn, COUNT(*) 
FROM books 
GROUP BY isbn 
HAVING COUNT(*) > 1;
```

如果查询结果为空，说明没有重复值，可以安全地创建唯一索引。

### 4.3.创建表时指定唯一索引

在创建表的时候，可以直接在 `CREATE TABLE` 语句中使用 `UNIQUE` 关键字来创建唯一索引。

**基本语法**：


```sql
CREATE TABLE table_name (
  column1 data_type,
  column2 data_type,
  ...,
  CONSTRAINT constraint_name UNIQUE (column1 [ASC|DESC], column2 [ASC|DESC], ...)
);
```

或者更简洁的方式，直接在列定义后添加 `UNIQUE`：


```sql
CREATE TABLE table_name (
  column1 data_type,
  column2 data_type UNIQUE,
  ...
);
```

**参数说明**：

- `CREATE TABLE`：创建表的关键字
- `table_name`：要创建的表名
- `(column1 data_type, ...)`：定义表的列和数据类型
- `CONSTRAINT`：用于添加约束的关键字
- `constraint_name`：约束名称
- `UNIQUE (column1, column2, ...)`：指定要创建唯一索引的列

**示例：创建表时同时创建唯一索引**

创建一个图书表，并在 ISBN 列上创建唯一索引：


```sql
CREATE TABLE books (
  id INT PRIMARY KEY,
  title VARCHAR(200),
  isbn VARCHAR(20),
  author VARCHAR(100),
  CONSTRAINT uk_book_isbn UNIQUE (isbn)
);
```

**示例：使用列级约束创建唯一索引**

也可以直接在列定义后添加 `UNIQUE` 关键字：


```sql
CREATE TABLE books (
  id INT PRIMARY KEY,
  title VARCHAR(200),
  isbn VARCHAR(20) UNIQUE,
  author VARCHAR(100)
);
```

这种方式更简洁，但索引名称会由 MySQL 自动生成。

**示例：创建多列唯一索引**

如果希望确保两列的组合值是唯一的，可以在创建表时定义：


```sql
CREATE TABLE book_reviews (
  id INT PRIMARY KEY,
  book_id INT,
  user_id INT,
  review_text TEXT,
  CONSTRAINT uk_book_user UNIQUE (book_id, user_id)
);
```

这个例子确保每个用户对每本书只能有一条评论。

## 5.使用 ALTER 命令管理索引

`ALTER TABLE` 命令是一个非常强大的工具，可以用来添加、修改和删除索引。本节将介绍如何使用 `ALTER TABLE` 命令来管理各种类型的索引。

### 5.1.添加索引的四种方式

使用 `ALTER TABLE` 命令可以通过四种不同的方式添加索引：

**方式一：添加普通索引**


```sql
ALTER TABLE table_name 
ADD INDEX index_name (column_list);
```

**方式二：添加唯一索引**


```sql
ALTER TABLE table_name 
ADD UNIQUE index_name (column_list);
```

**方式三：添加主键**


```sql
ALTER TABLE table_name 
ADD PRIMARY KEY (column_list);
```

**方式四：添加全文索引**


```sql
ALTER TABLE table_name 
ADD FULLTEXT index_name (column_list);
```

**各方式说明**：

| 方式     | 关键字        | 用途                       | 是否允许重复值     |
| -------- | ------------- | -------------------------- | ------------------ |
| 普通索引 | `INDEX`       | 提高查询性能               | 允许               |
| 唯一索引 | `UNIQUE`      | 提高查询性能并保证唯一性   | 不允许（NULL除外） |
| 主键     | `PRIMARY KEY` | 唯一标识每一行，不能为NULL | 不允许             |
| 全文索引 | `FULLTEXT`    | 用于全文搜索               | 允许               |

### 5.2.添加普通索引示例

假设我们有一个名为 `articles` 的表，现在要为标题列添加普通索引：


```sql
ALTER TABLE articles 
ADD INDEX idx_title (title);
```

这个语句在 `articles` 表的 `title` 列上创建了一个名为 `idx_title` 的普通索引。

### 5.3.添加唯一索引示例

为文章表的 URL 列添加唯一索引，确保每篇文章的 URL 都是唯一的：


```sql
ALTER TABLE articles 
ADD UNIQUE idx_url (url);
```

### 5.4.添加全文索引示例

全文索引主要用于文本搜索，适用于 `CHAR`、`VARCHAR` 或 `TEXT` 类型的列：


```sql
ALTER TABLE articles 
ADD FULLTEXT idx_content (content);
```

### 5.5.删除索引

使用 `ALTER TABLE` 命令的 `DROP INDEX` 子句可以删除索引：

**基本语法**：


```sql
ALTER TABLE table_name 
DROP INDEX index_name;
```

**示例：删除索引**

删除 `articles` 表上的 `idx_title` 索引：


```sql
ALTER TABLE articles 
DROP INDEX idx_title;
```

**注意事项**：

- 删除索引不会影响表中的数据
- 如果索引不存在，执行删除命令会产生错误
- 删除索引后，相关的查询性能可能会下降

## 6.使用 ALTER 命令管理主键

主键是表中唯一标识每一行数据的列或列组合。主键的值必须唯一且不能为 NULL。

### 6.1.添加主键

使用 `ALTER TABLE` 命令可以为一个已存在的表添加主键。

**基本语法**：


```sql
ALTER TABLE table_name 
ADD PRIMARY KEY (column_list);
```

**重要前提**：

- 主键列必须设置为 `NOT NULL`
- 主键列的值必须唯一
- 如果表中已经有数据，需要先确保数据满足上述条件

**示例：添加主键**

假设我们有一个名为 `comments` 的表，现在要为 `id` 列添加主键：

**步骤一：确保列不为空**


```sql
ALTER TABLE comments 
MODIFY id INT NOT NULL;
```

**步骤二：添加主键**


```sql
ALTER TABLE comments 
ADD PRIMARY KEY (id);
```

**示例：为多列添加复合主键**

如果希望使用多列组合作为主键：


```sql
ALTER TABLE order_items 
ADD PRIMARY KEY (order_id, product_id);
```

这个例子中，`order_id` 和 `product_id` 的组合作为主键，确保每个订单中的每个商品只能出现一次。

### 6.2.删除主键

使用 `ALTER TABLE` 命令可以删除表的主键。

**基本语法**：


```sql
ALTER TABLE table_name 
DROP PRIMARY KEY;
```

**示例：删除主键**

删除 `comments` 表的主键：


```sql
ALTER TABLE comments 
DROP PRIMARY KEY;
```

**注意事项**：

- 删除主键时只需指定 `PRIMARY KEY`，不需要指定名称
- 如果主键列上有自增属性（AUTO_INCREMENT），删除主键前可能需要先移除自增属性
- 删除主键不会删除数据，只是移除了主键约束

**示例：删除带自增属性的主键**

如果主键列有自增属性，需要先移除：


```sql
-- 先移除自增属性
ALTER TABLE comments 
MODIFY id INT NOT NULL;

-- 然后删除主键
ALTER TABLE comments 
DROP PRIMARY KEY;
```

## 7.查看索引信息

在实际使用中，我们经常需要查看表中已经创建的索引信息，以便了解索引的使用情况和优化查询性能。

### 7.1.使用 SHOW INDEX 命令

`SHOW INDEX` 命令可以显示表中所有索引的详细信息。

**基本语法**：


```sql
SHOW INDEX FROM table_name;
```

或者使用：


```sql
SHOW INDEXES FROM table_name;
```

**格式化输出**：

在 MySQL 命令行中，可以使用 `\G` 来格式化输出，让信息更易读：


```sql
SHOW INDEX FROM table_name\G
```

**参数说明**：

- `SHOW INDEX`：显示索引信息的关键字
- `FROM table_name`：指定要查看索引信息的表名
- `\G`：格式化输出，每行显示一个字段（仅在命令行中有效）

### 7.2.查看索引信息示例

假设我们有一个名为 `products` 的表，查看它的索引信息：


```sql
SHOW INDEX FROM products;
```

**输出示例**：

| Table    | Non_unique | Key_name | Seq_in_index | Column_name | Collation | Cardinality | Sub_part | Packed | Null | Index_type | Comment |
| -------- | ---------- | -------- | ------------ | ----------- | --------- | ----------- | -------- | ------ | ---- | ---------- | ------- |
| products | 0          | PRIMARY  | 1            | id          | A         | 1000        | NULL     | NULL   |      | BTREE      |         |
| products | 1          | idx_name | 1            | name        | A         | 950         | NULL     | NULL   | YES  | BTREE      |         |
| products | 0          | idx_code | 1            | code        | A         | 1000        | NULL     | NULL   | YES  | BTREE      |         |

### 7.3.索引信息字段说明

`SHOW INDEX` 命令输出的各个字段含义如下：

| 字段名         | 说明                                                         |
| -------------- | ------------------------------------------------------------ |
| `Table`        | 表名                                                         |
| `Non_unique`   | 是否为唯一索引。0 表示唯一索引，1 表示非唯一索引             |
| `Key_name`     | 索引名称。PRIMARY 表示主键                                   |
| `Seq_in_index` | 索引中的列序号，从 1 开始。对于组合索引，这个值表示列在索引中的位置 |
| `Column_name`  | 索引列的名称                                                 |
| `Collation`    | 列的排序方式。A 表示升序，NULL 表示未排序                    |
| `Cardinality`  | 索引中唯一值的数量估计值。这个值越大，索引的选择性越高       |
| `Sub_part`     | 索引的前缀长度。如果整个列都被索引，则为 NULL                |
| `Packed`       | 索引的打包方式                                               |
| `Null`         | 列是否允许 NULL 值                                           |
| `Index_type`   | 索引类型，通常是 BTREE                                       |
| `Comment`      | 索引的注释信息                                               |

### 7.6.从 INFORMATION_SCHEMA 查询索引信息

除了使用 `SHOW INDEX` 命令，还可以从 `INFORMATION_SCHEMA` 系统数据库中查询索引信息：


```sql
SELECT 
    TABLE_NAME,
    INDEX_NAME,
    COLUMN_NAME,
    SEQ_IN_INDEX,
    NON_UNIQUE
FROM 
    INFORMATION_SCHEMA.STATISTICS
WHERE 
    TABLE_SCHEMA = 'your_database_name'
    AND TABLE_NAME = 'your_table_name'
ORDER BY 
    INDEX_NAME, SEQ_IN_INDEX;
```

这种方式可以更灵活地筛选和排序索引信息。

## 8.实践建议

### 8.1.索引命名规范

建议使用统一的命名规范来命名索引，这样便于管理和识别：

- **普通索引**：使用 `idx_` 前缀，如 `idx_product_name`
- **唯一索引**：使用 `uk_` 或 `uq_` 前缀，如 `uk_user_email`
- **主键**：通常使用 `PRIMARY`（系统默认）
- **全文索引**：使用 `ft_` 前缀，如 `ft_article_content`

### 8.2.何时创建索引

**应该创建索引的情况**：

- 经常用于 `WHERE` 子句的列
- 经常用于 `ORDER BY` 和 `GROUP BY` 的列
- 经常用于表连接的列（外键）
- 数据量较大的表

**不应该创建索引的情况**：

- 数据量很小的表（通常少于 1000 行）
- 很少用于查询条件的列
- 频繁更新的列（会降低更新性能）
- 包含大量 NULL 值的列

### 8.3.组合索引的使用

创建组合索引时，需要注意列的顺序：

- **最左前缀原则**：组合索引 `(col1, col2, col3)` 可以用于查询 `col1`、`(col1, col2)` 或 `(col1, col2, col3)`，但不能用于只查询 `col2` 或 `col3`
- **选择性高的列放在前面**：选择性高的列（唯一值多的列）应该放在组合索引的前面

**示例**：


```sql
-- 好的组合索引：选择性高的列在前
CREATE INDEX idx_user_status_date ON orders (user_id, status, order_date);

-- 这个索引可以用于以下查询：
-- WHERE user_id = 123
-- WHERE user_id = 123 AND status = 'paid'
-- WHERE user_id = 123 AND status = 'paid' AND order_date > '2024-01-01'
```

### 8.4.定期检查和优化索引

- **定期查看索引使用情况**：使用 `SHOW INDEX` 查看索引信息
- **分析查询性能**：使用 `EXPLAIN` 命令分析查询是否使用了索引
- **删除未使用的索引**：如果某个索引很少被使用，考虑删除它以节省空间和提高更新性能

### 8.5.索引维护注意事项

- **创建索引时**：在数据量大的表上创建索引可能需要较长时间，建议在业务低峰期进行
- **删除索引时**：删除索引操作通常很快，但删除后相关查询的性能可能会下降
- **修改索引时**：MySQL 不支持直接修改索引，需要先删除再重新创建
