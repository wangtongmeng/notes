## 1.引言

在数据管理过程中，删除不需要的数据是常见的操作。删除数据可以帮助清理过时记录、移除错误信息，或者释放存储空间。

本文将从删除数据的基础概念开始，逐步介绍如何使用 `DELETE` 语句删除 MySQL 数据表中的记录，包括基本语法、条件删除、批量删除，以及如何通过命令行来执行删除操作。

## 2.删除数据基础概念

### 2.1.什么是删除数据？

删除数据是指从数据表中移除记录的操作。通过删除操作，可以永久性地移除表中不再需要的行，但不会影响表的结构。

**类比理解**：

- 数据表就像是一个 Excel 表格
- 删除数据就像是在表格中删除某一行
- 删除后，该行的数据就永久消失了（除非有备份）
- 表格的列结构（表头）不会改变

### 2.2.删除数据的前提条件

在执行删除操作之前，需要满足以下条件：

- **数据表已存在**：必须先创建好数据表，才能删除其中的数据
- **表中有数据**：如果表中没有数据，删除操作不会影响任何记录
- **已选择数据库**：需要先使用 `USE` 语句选择要操作的数据库
- **明确删除条件**：通常需要使用 `WHERE` 子句指定要删除哪些记录

### 2.3.删除数据的特点

- **可以删除单条记录**：通过精确条件删除一条记录
- **可以批量删除**：一次删除多条符合条件的记录
- **可以删除所有记录**：不使用 `WHERE` 子句时，会删除表中的所有数据
- **表结构保持不变**：删除数据不会影响表的列定义和结构
- **操作不可逆**：删除的数据无法直接恢复（除非有备份）

## 3.DELETE 语句基本语法

### 3.1.基本语法格式

删除 MySQL 数据表中的数据使用 `DELETE FROM` 语句，基本语法如下：


```sql
DELETE FROM table_name
WHERE condition;
```

**参数说明**：

- `table_name`：要删除数据的表名
- `WHERE condition`：可选的筛选条件，指定要删除的行。如果省略 `WHERE` 子句，将删除表中的所有行

### 3.2.语法要点

**重要提示**：

- `WHERE` 子句是可选的，但强烈建议使用
- 如果不指定 `WHERE` 条件，将删除表中的所有记录
- 删除操作是永久性的，执行前请仔细确认
- 可以使用各种条件表达式来精确控制要删除的记录

### 3.3.删除操作的执行流程

1. **连接数据库**：确保已连接到正确的数据库
2. **选择表**：指定要删除数据的表名
3. **设置条件**：使用 `WHERE` 子句指定删除条件（可选）
4. **执行删除**：MySQL 会找到所有符合条件的记录并删除
5. **返回结果**：显示受影响的行数

## 4.DELETE 语句使用示例

### 4.1.删除符合条件的单条记录

**场景**：删除员工表中工号为 1005 的员工记录


```sql
DELETE FROM employees
WHERE employee_id = 1005;
```

**说明**：

- `employees` 是员工表
- `employee_id = 1005` 是删除条件
- 只会删除工号等于 1005 的那一条记录

**执行结果示例**：


```python
Query OK, 1 row affected (0.15 sec)
```

### 4.2.删除符合条件的多条记录

**场景**：删除产品表中所有已下架的产品（状态为 'discontinued'）


```sql
DELETE FROM products
WHERE status = 'discontinued';
```

**说明**：

- `products` 是产品表
- `status = 'discontinued'` 是删除条件
- 会删除所有状态为 'discontinued' 的产品记录

**执行结果示例**：


```python
Query OK, 23 rows affected (0.28 sec)
```

### 4.3.使用多个条件删除

**场景**：删除订单表中 2022 年之前且金额小于 100 的订单


```sql
DELETE FROM orders
WHERE order_date < '2022-01-01'
  AND total_amount < 100;
```

**说明**：

- 使用 `AND` 连接多个条件
- 只有同时满足两个条件的记录才会被删除
- `order_date < '2022-01-01'` 表示日期在 2022 年之前
- `total_amount < 100` 表示金额小于 100

### 4.4.使用 IN 子句批量删除

**场景**：删除用户表中指定 ID 列表的用户


```sql
DELETE FROM users
WHERE user_id IN (101, 102, 103, 105, 108);
```

**说明**：

- `IN` 子句用于指定多个值
- 会删除 `user_id` 在列表中的所有记录
- 适合删除多个不连续的记录

### 4.5.使用 LIKE 模式匹配删除

**场景**：删除日志表中所有以 'test_' 开头的临时日志


```sql
DELETE FROM logs
WHERE log_name LIKE 'test_%';
```

**说明**：

- `LIKE` 用于模式匹配
- `'test_%'` 表示以 'test_' 开头的字符串
- `%` 是通配符，表示任意字符

### 4.6.删除所有记录

**场景**：清空临时表中的所有数据


```sql
DELETE FROM temp_table;
```

**说明**：

- 没有 `WHERE` 子句，会删除表中的所有记录
- 表结构保持不变，只是数据被清空
- **警告**：此操作会删除所有数据，请谨慎使用

**执行结果示例**：


```python
Query OK, 156 rows affected (0.42 sec)
```

### 4.7.使用子查询删除

**场景**：删除评论表中所有没有对应文章存在的评论


```sql
DELETE FROM comments
WHERE article_id NOT IN (
    SELECT article_id
    FROM articles
);
```

**说明**：

- 子查询用于查找 `articles` 表中存在的所有 `article_id`
- `NOT IN` 表示不在子查询结果中的记录
- 会删除所有 `article_id` 在 `articles` 表中不存在的评论

**另一种写法**：


```sql
DELETE FROM comments
WHERE article_id NOT EXISTS (
    SELECT 1
    FROM articles
    WHERE articles.article_id = comments.article_id
);
```

## 5.从命令行删除数据

### 5.1.连接到 MySQL

首先需要连接到 MySQL 服务器：


```bash
mysql -u root -p
```

输入密码后，会看到 `mysql>` 提示符。

### 5.2.选择数据库

使用 `USE` 语句选择要操作的数据库：


```sql
mysql> USE company_db;
Database changed
```

### 5.3.执行删除操作

**示例：删除部门表中部门编号为 10 的记录**


```sql
mysql> DELETE FROM departments
    -> WHERE dept_id = 10;
Query OK, 1 row affected (0.18 sec)
```

**说明**：

- `departments` 是部门表
- `dept_id = 10` 是删除条件
- `Query OK, 1 row affected` 表示成功删除了 1 条记录
- `(0.18 sec)` 表示操作耗时 0.18 秒

### 5.4.验证删除结果

删除后，可以使用 `SELECT` 语句验证删除是否成功：


```sql
mysql> SELECT * FROM departments WHERE dept_id = 10;
Empty set (0.00 sec)
```

`Empty set` 表示没有找到记录，说明删除成功。

### 5.5.查看受影响的行数

MySQL 会返回受影响的行数，可以通过这个信息确认删除操作的结果：


```sql
mysql> DELETE FROM products
    -> WHERE stock_quantity = 0;
Query OK, 15 rows affected (0.25 sec)
```

这表示成功删除了 15 条库存为 0 的产品记录。

## 6.删除操作的注意事项

### 6.1.备份重要数据

在执行删除操作前，建议先备份数据：


```sql
-- 创建备份表
CREATE TABLE employees_backup AS
SELECT * FROM employees;

-- 或者导出数据
mysqldump -u root -p company_db employees > employees_backup.sql
```

### 6.2.使用事务

对于重要的删除操作，建议使用事务：


```sql
START TRANSACTION;

DELETE FROM orders WHERE order_date < '2020-01-01';

-- 检查删除结果
SELECT COUNT(*) FROM orders;

-- 如果结果正确，提交事务
COMMIT;

-- 如果结果不正确，回滚事务
-- ROLLBACK;
```

### 6.3.先查询再删除

在执行删除前，先用 `SELECT` 查询确认要删除的记录：


```sql
-- 先查看要删除的记录
SELECT * FROM orders
WHERE order_date < '2020-01-01';

-- 确认无误后，再执行删除
DELETE FROM orders
WHERE order_date < '2020-01-01';
```

这样可以避免误删数据，确保删除操作的安全性。

### 6.4.限制删除数量

如果需要限制删除的记录数量，可以使用 `LIMIT` 子句：


```sql
-- 只删除前 10 条记录
DELETE FROM logs
WHERE log_level = 'DEBUG'
LIMIT 10;
```

**注意**：`LIMIT` 子句在 `DELETE` 语句中的使用有限制，某些 MySQL 版本可能不支持。

### 6.5.外键约束的影响

如果表之间存在外键关系，删除操作可能会受到约束：


```sql
-- 如果存在外键约束，可能需要先删除子表记录
DELETE FROM order_items
WHERE order_id = 1001;

-- 然后再删除主表记录
DELETE FROM orders
WHERE order_id = 1001;
```

## 7.DELETE 与其他语句的区别

### 7.1.DELETE vs TRUNCATE

`DELETE` 和 `TRUNCATE` 都可以清空表，但有以下区别：

| 特性              | DELETE | TRUNCATE         |
| ----------------- | ------ | ---------------- |
| 可以带 WHERE 条件 | 是     | 否               |
| 可以回滚          | 是     | 否（某些情况下） |
| 删除速度          | 较慢   | 较快             |
| 重置自增列        | 否     | 是               |
| 触发触发器        | 是     | 否               |

**示例对比**：


```sql
-- DELETE：可以带条件，删除后自增列不会重置
DELETE FROM users WHERE status = 'inactive';

-- TRUNCATE：清空所有数据，自增列会重置
TRUNCATE TABLE users;
```

### 7.2.DELETE vs DROP

`DELETE` 只删除数据，`DROP` 会删除整个表：


```sql
-- DELETE：只删除数据，表结构保留
DELETE FROM users;

-- DROP：删除整个表（包括结构和数据）
DROP TABLE users;
```

## 8.实践建议

### 8.1.删除前的检查清单

在执行删除操作前，建议检查以下事项：

1. **确认数据库和表名正确**


   ```sql
   SELECT DATABASE(); SHOW TABLES;
   ```

2. **先查询要删除的数据**


   ```sql
   SELECT * FROM table_name WHERE condition;
   ```

3. **备份重要数据**


   ```sql
   CREATE TABLE backup_table AS SELECT * FROM table_name;
   ```

4. **使用事务保护**


   ```sql
   START TRANSACTION; DELETE FROM table_name WHERE condition; -- 检查结果后再决定 COMMIT 或 ROLLBACK
   ```

### 8.2.性能优化建议

对于大量数据的删除操作，可以考虑以下优化：

1. **分批删除**：避免一次性删除大量数据


   ```sql
   DELETE FROM logs WHERE log_date < '2020-01-01' LIMIT 1000; -- 重复执行直到删除完成
   ```

2. **在低峰期执行**：避免影响正常业务

3. **添加索引**：确保 WHERE 条件中的字段有索引

4. **使用临时表**：对于复杂的删除条件，可以先筛选到临时表


   ```sql
   CREATE TEMPORARY TABLE temp_ids AS SELECT id FROM orders WHERE order_date < '2020-01-01';  DELETE FROM orders WHERE id IN (SELECT id FROM temp_ids);
   ```

### 8.3.安全建议

1. **使用参数化查询**：防止 SQL 注入
2. **限制删除权限**：只给必要的用户删除权限
3. **启用审计日志**：记录所有删除操作
4. **定期备份**：确保数据可恢复
